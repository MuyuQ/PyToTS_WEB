## 🎯 为什么这个很重要

你在 Python 里用惯了灵活的函数：

```python
# 想传几个参数传几个
def process(*args, **kwargs):
    ...

# 想返回什么返回什么
def get_data():
    if condition:
        return {"status": "ok"}
    else:
        return None
```

转到 TypeScript 后，第一天就踩坑：

```typescript
// ❌ 编译报错：参数数量不匹配
process(1, 2, 3); // Expected 0 arguments, but got 3

// ❌ 类型推断可能不是你想要的
function get_data() {
  if (condition) {
    return { status: "ok" }; // 类型推断为 { status: string } | undefined
  }
}
```

刚开始觉得束手束脚，但慢慢发现：这些"束缚"让你少踩了很多坑...

## 💡 核心概念：函数是"合同"

**Python 函数**像"万能插座"——什么都能插，插错了通电才知道。

**TypeScript 函数**像"专用接口"——只接受匹配的插头，不匹配根本插不进去。

**函数签名**就是"合同"——规定了输入输出的规格：

```python
# Python：合同只是"建议"
def add(a: int, b: int) -> int:
    return a + b

add("1", "2")  # 运行时不会报错，但逻辑可能不是你想要的
```

```typescript
// TypeScript：合同是"法律"
function add(a: number, b: number): number {
  return a + b;
}

add("1", "2"); // ❌ 编译报错：类型不匹配
```

## 📚 代码对比

### 基本函数定义

```python
# Python：简洁但有坑
def add(a: int, b: int) -> int:
    return a + b
```

```typescript
// TypeScript：冗长但安全
function add(a: number, b: number): number {
  return a + b;
}

// 箭头函数写法（更简洁）
const add = (a: number, b: number): number => a + b;
```

### 默认参数

```python
# Python：默认参数很自然
def greet(name: str, greeting: str = "Hello") -> str:
    return f"{greeting}, {name}"

greet("Alice")           # "Hello, Alice"
greet("Bob", "Hi")      # "Hi, Bob"
```

```typescript
// TypeScript：语法相同
function greet(name: string, greeting: string = "Hello"): string {
  return `${greeting}, ${name}`;
}

greet("Alice"); // "Hello, Alice"
greet("Bob", "Hi"); // "Hi, Bob"
```

### 可变参数

```python
# Python：*args 接收任意数量的位置参数
def sum_all(*args: int) -> int:
    return sum(args)

sum_all(1, 2, 3)  # 6
sum_all()         # 0
```

```typescript
// TypeScript：...rest 参数（注意三个点）
function sumAll(...args: number[]): number {
  return args.reduce((a, b) => a + b, 0);
}

sumAll(1, 2, 3); // 6
sumAll(); // 0
```

**类比**：`*args` 和 `...rest` 就像"打包袋"——把散落的东西打包成一个数组。

### 关键字参数

```python
# Python：**kwargs 接收任意关键字参数
def create_user(name: str, **kwargs) -> dict:
    return {"name": name, **kwargs}

create_user("Alice", age=25, city="Beijing")
# {"name": "Alice", "age": 25, "city": "Beijing"}
```

```typescript
// TypeScript：没有 **kwargs，用对象参数代替
function createUser(name: string, options: Record<string, unknown>): object {
  return { name, ...options };
}

createUser("Alice", { age: 25, city: "Beijing" });
// { name: "Alice", age: 25, city: "Beijing" }
```

> **注意**：TypeScript 没有 `**kwargs`，必须显式定义对象参数。虽然麻烦，但类型更清晰。

### 函数重载

**类比**：函数重载像"变形金刚"——同一个名字，不同参数有不同行为。

```python
# Python：不支持函数重载，后定义的会覆盖前面的
def process(data: str) -> str:
    return data.upper()

def process(data: int) -> int:  # ❌ 覆盖了上面的
    return data * 2

# 只能通过类型判断
def process(data):
    if isinstance(data, str):
        return data.upper()
    elif isinstance(data, int):
        return data * 2
```

```typescript
// TypeScript：支持真正的函数重载
function process(data: string): string;
function process(data: number): number;
function process(data: string | number): string | number {
  if (typeof data === "string") {
    return data.toUpperCase();
  } else {
    return data * 2;
  }
}

process("hello"); // "HELLO"（返回 string）
process(5); // 10（返回 number）
```

## ⚡ 语法速查表

| 功能       | Python            | TypeScript                  |
| ---------- | ----------------- | --------------------------- |
| 函数定义   | `def func():`     | `function func(): {}`       |
| 箭头函数   | ❌ 不支持         | `const func = () => {}`     |
| 默认参数   | `def f(a=1):`     | `function f(a = 1) {}`      |
| 可变参数   | `*args`           | `...args`                   |
| 关键字参数 | `**kwargs`        | 用对象参数代替              |
| 可选参数   | ❌ 用默认参数模拟 | `function f(a?: number) {}` |
| 函数重载   | ❌ 不支持         | ✅ 支持                     |
| 返回值类型 | `-> int`          | `: number`                  |

## 💬 开发者对话

🐍 Python 开发者："TypeScript 函数参数太麻烦了！我想传什么传什么，干嘛要规定类型？"

🦀 TS 开发者："等你在生产环境踩过坑就知道了。上次有个函数期望 `number`，结果传了 `string`，运行时数据全乱了..."

🐍 "那 `*args` 和 `**kwargs` 呢？TypeScript 为什么没有？"

🦀 "`*args` 有，叫 `...rest`。但 `**kwargs` 确实没有，因为 TypeScript 要求类型明确。你可以用对象参数代替，虽然多写几行，但 IDE 能自动补全。"

🐍 "可选参数呢？Python 用默认参数就行了。"

🦀 "TypeScript 有专门的 `?` 语法，更直观。`function f(a?: number)` 表示 `a` 可以不传。"

```typescript
// 可选参数：? 表示可以不传
function greet(name: string, greeting?: string): string {
  return `${greeting ?? "Hello"}, ${name}`;
}

greet("Alice"); // "Hello, Alice"
greet("Bob", "Hi"); // "Hi, Bob"
```

🐍 "那函数重载有什么用？"

🦀 "让同一个函数根据参数类型返回不同类型。比如上面的 `process` 函数，传 `string` 返回 `string`，传 `number` 返回 `number`。Python 只能靠类型判断，TypeScript 编译器就能帮你检查。"

## ⚠️ 常见陷阱

### 1. 参数数量必须匹配

```typescript
// ❌ 错误：参数数量不匹配
function add(a: number, b: number): number {
  return a + b;
}

add(1); // Expected 2 arguments, but got 1
add(1, 2, 3); // Expected 2 arguments, but got 3
```

```python
# Python 允许这样（虽然不推荐）
def add(a: int, b: int) -> int:
    return a + b

add(1)  # TypeError: missing 1 required positional argument
```

### 2. 可选参数必须在必选参数后面

```typescript
// ❌ 错误：可选参数不能放在前面
function greet(greeting?: string, name: string): string {
  return `${greeting}, ${name}`;
}

// ✅ 正确：可选参数放后面
function greet(name: string, greeting?: string): string {
  return `${greeting ?? "Hello"}, ${name}`;
}
```

### 3. `for...in` vs `for...of` 的坑

```typescript
const arr = ["a", "b", "c"];

// ❌ for...in 遍历的是索引（字符串）
for (const i in arr) {
  console.log(i); // "0", "1", "2"（字符串！）
  console.log(arr[i]); // "a", "b", "c"
}

// ✅ for...of 遍历的是值
for (const v of arr) {
  console.log(v); // "a", "b", "c"
}
```

**类比**：`for...in` 像翻书页码（索引），`for...of` 像看书内容（值）。

## 🤔 试试看

### 预测输出

下面这段代码会输出什么？

```typescript
function process(data: string | number): string | number {
  if (typeof data === "string") {
    return data.toUpperCase();
  }
  return data * 2;
}

const result1 = process("hello");
const result2 = process(5);
const result3 = process(3 as any); // 强制绕过类型检查

console.log(typeof result1, result1);
console.log(typeof result2, result2);
console.log(typeof result3, result3);
```

<details>
<summary>点击查看答案</summary>

```
string HELLO
number 10
number 6
```

解析：

- `result1` 传了 `string`，进入 `if` 分支，返回大写字符串
- `result2` 传了 `number`，进入 `else` 分支，返回数字翻倍
- `result3` 用 `as any` 强制绕过类型检查，但运行时行为正常

**注意**：虽然返回类型是 `string | number`，但 TypeScript 无法根据输入推断具体返回类型。如果要实现精确的类型推断，需要用函数重载。

</details>

### 找错误

下面这段代码有什么问题？

```typescript
function createUser(name: string, options?: object) {
  return {
    name,
    ...options, // 展开 options
  };
}

const user = createUser("Alice", { age: 25 });
console.log(user.age); // 想访问 age 属性
```

<details>
<summary>点击查看答案</summary>

问题：`options?: object` 类型太宽泛，TypeScript 不知道 `options` 里有什么属性。

```typescript
// ❌ 错误：Property 'age' does not exist on type...
console.log(user.age);
```

修正方案：

```typescript
// 方案 1：定义具体的接口
interface UserOptions {
  age?: number;
  city?: string;
}

function createUser(name: string, options?: UserOptions) {
  return { name, ...options };
}

const user = createUser("Alice", { age: 25 });
console.log(user.age); // ✅ 正确：类型推断为 number | undefined

// 方案 2：使用泛型（更灵活）
function createUser<T extends object>(name: string, options?: T) {
  return { name, ...options } as { name: string } & T;
}

const user = createUser("Alice", { age: 25 });
console.log(user.age); // ✅ 正确：25
```

**教训**：`object` 类型只是说"这是一个对象"，但不知道对象里有什么。要用接口或泛型来明确类型。

</details>

### 代码翻译

将以下 Python 函数转换为 TypeScript：

```python
def format_message(title: str, body: str, urgent: bool = False) -> str:
    prefix = "[URGENT] " if urgent else ""
    return f"{prefix}{title}: {body}"
```

<details>
<summary>点击查看答案</summary>

```typescript
function formatMessage(title: string, body: string, urgent: boolean = false): string {
  const prefix = urgent ? "[URGENT] " : "";
  return `${prefix}${title}: ${body}`;
}

// 或者用可选参数 + 默认值
function formatMessage(title: string, body: string, urgent?: boolean): string {
  const prefix = urgent ? "[URGENT] " : "";
  return `${prefix}${title}: ${body}`;
}
```

**记忆口诀**：默认参数直接写 `= value`，可选参数写 `?` 然后在函数里处理 `undefined`。

</details>

## 📝 优势对比

| 特性       | Python     | TypeScript | 说明                  |
| ---------- | ---------- | ---------- | --------------------- |
| 默认参数   | ✓          | ✓          | 语法相同              |
| 可变参数   | `*args`    | `...args`  | 都打包成数组          |
| 关键字参数 | `**kwargs` | 对象参数   | TS 需要显式定义类型   |
| 函数重载   | ✗          | ✓          | TS 支持真正的重载     |
| 箭头函数   | ✗          | ✓          | TS 有更简洁的函数写法 |
| 可选参数   | 用默认参数 | `?` 语法   | TS 语法更明确         |
| 类型推断   | 运行时检查 | 编译时检查 | TS 能提前发现错误     |

## 🎓 面试实战

1. **TypeScript 箭头函数和普通函数的 `this` 有什么区别？**

<details>
<summary>点击查看答案</summary>

箭头函数没有自己的 `this`，继承外部上下文的 `this`。普通函数的 `this` 取决于调用方式。

```typescript
const obj = {
  name: "Alice",

  // ❌ 箭头函数：this 指向外层（可能是 undefined）
  greetArrow: () => {
    console.log(this.name); // undefined（严格模式）
  },

  // ✅ 普通函数：this 指向调用对象
  greetNormal() {
    console.log(this.name); // "Alice"
  },
};

obj.greetArrow(); // undefined
obj.greetNormal(); // "Alice"
```

**记忆口诀**：箭头函数锁定 `this`，普通函数动态 `this`。

</details>

2. **如何在 TypeScript 中实现函数重载？**

<details>
<summary>点击查看答案</summary>

函数重载需要：先写多个签名，再写一个通用实现。

```typescript
// 重载签名
function format(input: string): string;
function format(input: number): string;
function format(input: string | number): string {
  if (typeof input === "string") {
    return input.toUpperCase();
  }
  return input.toFixed(2);
}

format("hello"); // "HELLO"（返回 string）
format(3.14159); // "3.14"（返回 string）
```

**注意**：实现签名（最后一行）的参数类型必须兼容所有重载签名。

</details>

3. **可选参数和默认参数有什么区别？**

<details>
<summary>点击查看答案</summary>

```typescript
// 可选参数：不传时是 undefined
function greet1(name: string, greeting?: string): string {
  return `${greeting ?? "Hello"}, ${name}`;
}

// 默认参数：不传时使用默认值
function greet2(name: string, greeting: string = "Hello"): string {
  return `${greeting}, ${name}`;
}

// 区别：
greet1("Alice"); // greeting 是 undefined
greet1("Alice", undefined); // greeting 是 undefined

greet2("Alice"); // greeting 是 "Hello"
greet2("Alice", undefined); // greeting 是 "Hello"（默认值生效）
```

**关键点**：传 `undefined` 时，默认参数会用默认值，可选参数就是 `undefined`。

</details>
