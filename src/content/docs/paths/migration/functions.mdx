# 函数与参数

## 场景与问题

函数是代码复用的基础单元。Python 和 TypeScript 都支持函数定义，但语法和特性有所不同。

## Python 回顾

```python
# 基本函数
def add(a: int, b: int) -> int:
    return a + b

# 默认参数
def greet(name: str, greeting: str = "Hello") -> str:
    return f"{greeting}, {name}"

# 可变参数
def sum_all(*args: int) -> int:
    return sum(args)

# 关键字参数
def create_user(name: str, **kwargs) -> dict:
    return {"name": name, **kwargs}
```

## TypeScript 等价写法

```typescript
// 基本函数
function add(a: number, b: number): number {
    return a + b;
}

// 默认参数
function greet(name: string, greeting: string = "Hello"): string {
    return `${greeting}, ${name}`;
}

// 可变参数 (rest 参数)
function sumAll(...args: number[]): number {
    return args.reduce((a, b) => a + b, 0);
}

// 对象参数
function createUser(name: string, options: Record<string, unknown>): object {
    return { name, ...options };
}
```

## 差异与常见陷阱

1. **参数默认值**：语法相同，但 TypeScript 更严格
2. **可变参数**：Python 用 `*args`，TypeScript 用 `...args`
3. **关键字参数**：TypeScript 没有直接等价物，使用对象参数代替
4. **函数重载**：TypeScript 支持函数重载，Python 不支持

## 优势对比

| 特性 | Python | TypeScript |
|------|--------|------------|
| 默认参数 | ✓ | ✓ |
| 可变参数 | `*args` | `...args` |
| 关键字参数 | `**kwargs` | 对象参数 |
| 函数重载 | ✗ | ✓ |
| 箭头函数 | ✗ | ✓ |

## 练习

将以下 Python 函数转换为 TypeScript：

```python
def format_message(title: str, body: str, urgent: bool = False) -> str:
    prefix = "[URGENT] " if urgent else ""
    return f"{prefix}{title}: {body}"
```
