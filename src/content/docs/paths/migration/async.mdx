## 🎯 为什么这个很重要

想象你在餐厅点外卖...

**同步方式（Python 默认）**：
你站在柜台前等，外卖不到不吃午饭。这期间你什么都做不了，只能盯着出餐口发呆。

**异步方式（TypeScript 原生）**：
你点了外卖，拿了号码牌，回工位继续写代码。外卖好了，餐厅打电话（Promise resolved），你去取餐。如果外卖卖完了（Promise rejected），你选择改吃泡面或退款。

这就是同步和异步的核心区别：**等待的时候能不能做别的事**。

Python 有 `asyncio` 库支持异步，但默认是同步执行。JavaScript/TypeScript 天生异步，几乎所有 I/O 操作都是异步的。

---
## 💡 核心概念：用类比理解

| 概念               | 类比           | 说明                                                |
| ------------------ | -------------- | --------------------------------------------------- |
| **Promise**        | 外卖订单       | 已下单 pending → 送到了 fulfilled → 送不了 rejected |
| **async/await**    | 等外卖的标记   | async 函数可以"暂停"等外卖送到                      |
| **回调函数**       | 留电话号码     | "外卖到了打这个电话"                                |
| **Event Loop**     | 餐厅服务员     | 轮流服务每桌客人，不让一桌等太久                    |
| **.then()**        | 订单后续操作   | 送到了就吃，送不了就投诉                            |
| **Promise.all()**  | 同时点多家外卖 | 等所有外卖都到了再开饭                              |
| **Promise.race()** | 谁先到吃谁     | 多家外卖比赛，先到的赢                              |

## 📚 Python 回顾

```python
import asyncio

# 异步函数定义（外卖订单）
async def fetch_data(url: str) -> str:
    await asyncio.sleep(1)  # 模拟网络请求（等外卖）
    return f"Data from {url}"

# 并发执行（同时点多家外卖）
async def main():
    tasks = [
        fetch_data("url1"),
        fetch_data("url2"),
    ]
    results = await asyncio.gather(*tasks)  # 等所有外卖送到
    return results

# 运行（启动事件循环）
asyncio.run(main())
```

**Python 异步要点**：

- `async def` 定义协程函数
- `await` 等待异步操作完成
- `asyncio.run()` 启动事件循环
- `asyncio.gather()` 并发执行多个协程

## 🔧 TypeScript 等价写法

```typescript
// 异步函数定义（外卖订单）
async function fetchData(url: string): Promise<string> {
  await new Promise((resolve) => setTimeout(resolve, 1000)); // 等外卖
  return `Data from ${url}`;
}

// 并发执行（同时点多家外卖）
async function main(): Promise<string[]> {
  const results = await Promise.all([fetchData("url1"), fetchData("url2")]);
  return results;
}

// 运行（自动启动事件循环）
main();
```

**TypeScript 异步要点**：

- `async function` 定义异步函数
- 返回类型是 `Promise<T>`
- `await` 等待 Promise 完成
- 事件循环自动运行，不需要手动启动
- `Promise.all()` 并发执行多个 Promise

## 📊 语法速查表

| 特性         | Python                            | TypeScript                          |
| ------------ | --------------------------------- | ----------------------------------- |
| 异步函数     | `async def func():`               | `async function func(): Promise<T>` |
| 等待结果     | `await coro()`                    | `await promise`                     |
| 并发执行     | `asyncio.gather(*tasks)`          | `Promise.all([...promises])`        |
| 竞速执行     | `asyncio.wait_for(coro, timeout)` | `Promise.race([...promises])`       |
| 启动事件循环 | `asyncio.run(main())`             | 自动运行                            |
| 错误处理     | `try/except`                      | `try/catch` 或 `.catch()`           |

## ⚠️ 差异与常见陷阱

### 1. 返回类型必须写

Python 的 `async def` 自动返回协程对象，TypeScript 必须写 `Promise<T>`：

```typescript
// ❌ 错误：忘记写 Promise
async function fetchUser(id: number): User {
  // Error: The return type of an async function must be a promise
  return await db.findUser(id);
}

// ✅ 正确
async function fetchUser(id: number): Promise<User> {
  return await db.findUser(id);
}
```

### 2. 事件循环自动运行

Python 需要手动启动事件循环，TypeScript 浏览器/Node 自动运行：

```python
# Python：必须显式启动
asyncio.run(main())
```

```typescript
// TypeScript：直接调用就行
main();
```

### 3. 忘记 await

这是最常见的错误，Promise 没有 await 就不会等待：

```typescript
async function bad() {
  const data = fetchData("url"); // ❌ 没有 await，data 是 Promise，不是结果
  console.log(data); // Promise { <pending> }
}

async function good() {
  const data = await fetchData("url"); // ✅ 有 await，data 是实际结果
  console.log(data); // "Data from url"
}
```

### 4. 错误处理方式不同

Python 用 `try/except`，TypeScript 可以用 `try/catch` 或 `.catch()`：

```typescript
// 方式 1：try/catch（推荐）
async function safeFetch() {
  try {
    const data = await fetchData("url");
    return data;
  } catch (error) {
    console.error("获取失败:", error);
    return null;
  }
}

// 方式 2：.catch() 链式调用
fetchData("url")
  .then((data) => console.log(data))
  .catch((error) => console.error(error));
```

### 5. Promise.all 一个失败全失败

`Promise.all` 只要有一个 Promise reject，整个就 reject：

```typescript
// 如果 fetchUser(2) 失败，整个 Promise.all 都失败
const users = await Promise.all([
  fetchUser(1), // 成功
  fetchUser(2), // 失败！
  fetchUser(3), // 成功
]);
// users 拿不到任何结果
```

如果需要"允许部分失败"，用 `Promise.allSettled`：

```typescript
const results = await Promise.allSettled([fetchUser(1), fetchUser(2), fetchUser(3)]);
// results = [
//   { status: "fulfilled", value: ... },
//   { status: "rejected", reason: ... },
//   { status: "fulfilled", value: ... }
// ]
```

## 💬 开发者对话

**Python 开发者**：我习惯了 `asyncio.run()`，TypeScript 不需要吗？

**TypeScript 开发者**：不需要。JavaScript 运行时（浏览器或 Node.js）自动维护事件循环。你直接调用 `main()` 就行，Promise 会自动调度执行。

**Python 开发者**：那 `.then()` 和 `await` 有什么区别？看起来都能处理异步结果？

**TypeScript 开发者**：

- `.then()` 是链式调用，适合简单操作
- `await` 是暂停等待，代码看起来像同步，更适合复杂逻辑

```typescript
// .then() 链式调用
fetchData("url")
  .then((data) => processData(data))
  .then((result) => saveResult(result))
  .catch((error) => handleError(error));

// await 同步风格（更易读）
try {
  const data = await fetchData("url");
  const result = processData(data);
  await saveResult(result);
} catch (error) {
  handleError(error);
}
```

**Python 开发者**：我明白了，`await` 让异步代码看起来像同步代码，可读性更好。

**TypeScript 开发者**：对。但记住，`await` 只能在 `async` 函数内部使用，这是硬性规定。

**Python 开发者**：`Promise.all` 和 `asyncio.gather` 一样吗？

**TypeScript 开发者**：基本一样，都是并发执行多个异步操作。但有个坑：`Promise.all` 只要有一个失败，整个就失败。如果需要"部分失败也继续"，用 `Promise.allSettled`。

**Python 开发者**：Python 的 `asyncio.gather` 可以用 `return_exceptions=True` 达到类似效果。

**TypeScript 开发者**：没错，思路一样。另外 `Promise.race` 返回最快的结果，`Promise.any` 返回第一个成功的。

### 预测输出

下面这段代码的输出顺序是什么？

```typescript
async function test() {
  console.log("1. 开始");

  const p1 = new Promise((resolve) => {
    console.log("2. 创建 Promise");
    resolve("3. Promise 结果");
  });

  console.log("4. 准备 await");

  const result = await p1;
  console.log(result);

  console.log("5. 结束");
}

test();
console.log("6. 主线程继续");
```

<details>
<summary>点击查看答案</summary>

输出顺序：

```
1. 开始
2. 创建 Promise
4. 准备 await
6. 主线程继续
3. Promise 结果
5. 结束
```

**关键点**：

- Promise 创建时立即执行（同步），输出 "2"
- `await` 会暂停当前函数，让出控制权
- "6. 主线程继续" 先执行，因为 test() 被暂停了
- Promise resolve 后，await 后面的代码才执行

</details>

### 找错误

下面这段代码有什么问题？

```typescript
async function fetchUsers() {
  const users = [];
  for (const id of [1, 2, 3]) {
    const user = fetchUser(id); // 获取用户数据
    users.push(user);
  }
  return users;
}

async function main() {
  const users = await fetchUsers();
  console.log(users[0].name); // 想打印第一个用户的名字
}
```

<details>
<summary>点击查看答案</summary>

**问题**：`fetchUser(id)` 前面没有 `await`，所以 `user` 是 Promise 对象，不是用户数据。

**结果**：

- `users` 是 `[Promise, Promise, Promise]`
- `users[0].name` 是 `undefined`（Promise 没有 name 属性）

**修复方式 1**：串行执行（一个接一个）

```typescript
async function fetchUsers() {
  const users = [];
  for (const id of [1, 2, 3]) {
    const user = await fetchUser(id); // 加上 await
    users.push(user);
  }
  return users;
}
```

**修复方式 2**：并行执行（同时发起）

```typescript
async function fetchUsers() {
  const promises = [1, 2, 3].map((id) => fetchUser(id));
  const users = await Promise.all(promises);
  return users;
}
```

</details>

## 📝 练习

将以下 Python 异步代码转换为 TypeScript：

```python
async def fetch_user(id: int) -> dict:
    await asyncio.sleep(0.5)
    return {"id": id, "name": f"User {id}"}

async def get_all_users() -> list:
    return await asyncio.gather(
        fetch_user(1),
        fetch_user(2),
        fetch_user(3),
    )
```

<details>
<summary>点击查看参考答案</summary>

```typescript
async function fetchUser(id: number): Promise<{ id: number; name: string }> {
  await new Promise((resolve) => setTimeout(resolve, 500));
  return { id, name: `User ${id}` };
}

async function getAllUsers(): Promise<{ id: number; name: string }[]> {
  return await Promise.all([fetchUser(1), fetchUser(2), fetchUser(3)]);
}
```

**注意点**：

- 返回类型必须写 `Promise<T>`
- `asyncio.sleep()` → `setTimeout` 包装成 Promise
- `asyncio.gather()` → `Promise.all()`
- TypeScript 字典类型用 `{ key: type }` 或 `Record<K, V>`

</details>

## 🎓 面试实战

### 问题 1：Promise.all 和 Promise.allSettled 有什么区别？

`Promise.all`：一个失败，整体失败。适合"要么全成功，要么全重来"的场景。

`Promise.allSettled`：等待所有 Promise 完成（无论成功失败），返回每个的结果状态。适合"收集所有结果，失败的也要知道"的场景。

### 问题 2：如何处理 Promise 的错误？

两种方式：

```typescript
// 方式 1：try/catch（推荐，代码更清晰）
async function handleRequest() {
  try {
    const data = await fetchData();
    return data;
  } catch (error) {
    console.error(error);
    throw error; // 或返回默认值
  }
}

// 方式 2：.catch() 链式调用
fetchData()
  .then((data) => processData(data))
  .catch((error) => console.error(error));
```

### 问题 3：async 函数返回什么？

async 函数总是返回 Promise，即使你 return 的是普通值：

```typescript
async function foo() {
  return 42; // 等价于 return Promise.resolve(42)
}

const result = foo(); // result 是 Promise<number>，不是 42
const value = await foo(); // value 才是 42
```

### 问题 4：为什么 await 必须在 async 函数内？

因为 `await` 会暂停函数执行，让出控制权给事件循环。只有在 async 函数（协程）中才能安全暂停和恢复。如果在普通函数中使用 await，会无法恢复执行。

顶层 await（Top-level await）是 ES2022 的新特性，允许在模块顶层直接使用 await，但本质上模块本身就是异步加载的。
