## 🎯 为什么这个很重要

你在 Python 里写类，`self` 总是明明白白。转到 TypeScript 后：

```typescript
class Counter {
  count = 0;
  increment() {
    setTimeout(function () {
      console.log(this.count); // undefined！
    }, 100);
  }
}
```

这是什么情况？`this` 怎么变成了 `undefined`？

还有这些问题：

- 构造函数里忘了初始化属性，类型检查报错
- `private` 和 `protected` 到底有什么区别？
- 接口和抽象类，什么时候用哪个？

类和对象是面向对象的基础，TypeScript 的类型系统让类更严谨，但也带来了新的学习曲线。理解这些差异，能让你写出更优雅、更安全的代码。

## 💡 用类比理解面向对象

把类和对象想象成建筑：

| 概念         | 类比         | 说明                                            |
| ------------ | ------------ | ----------------------------------------------- |
| **类**       | 建筑图纸     | 定义了房子的结构，但还不能住人                  |
| **对象**     | 建好的房子   | 按图纸建好，可以住进去（使用实例）              |
| **构造函数** | 施工队       | 按图纸把房子建起来，初始化各个属性              |
| **this**     | 房主         | 谁调用方法，this 就指向谁（但要注意调用方式！） |
| **接口**     | 房屋验收标准 | 规定了房子必须有哪些设施（空调、厨房...）       |
| **继承**     | 户型改良     | 在原有户型基础上加个阳台、改个窗户              |

Python 的 `self` 像是房主亲自来处理事务，明明白白。JavaScript/TypeScript 的 `this` 像是代理人，根据谁在说话，可能指向不同的人。

## Python 回顾

```python
class Person:
    # 类属性（所有实例共享）
    species = "Homo sapiens"

    # 构造方法
    def __init__(self, name: str, age: int):
        # 实例属性
        self.name = name
        self._age = age  # 约定：受保护属性
        self.__id = 123  # 名称修饰：私有属性

    # 实例方法
    def introduce(self) -> str:
        return f"I'm {self.name}, {self._age} years old"

    # 类方法
    @classmethod
    def create_anonymous(cls) -> "Person":
        return cls("Anonymous", 0)

    # 静态方法
    @staticmethod
    def is_adult(age: int) -> bool:
        return age >= 18

    # 属性装饰器
    @property
    def age(self) -> int:
        return self._age

    @age.setter
    def age(self, value: int) -> None:
        if value < 0:
            raise ValueError("Age cannot be negative")
        self._age = value

# 继承
class Student(Person):
    def __init__(self, name: str, age: int, grade: str):
        super().__init__(name, age)
        self.grade = grade

    def introduce(self) -> str:
        base = super().introduce()
        return f"{base}, Grade: {self.grade}"
```

## TypeScript 等价写法

```typescript
class Person {
  // 类属性
  static species = "Homo sapiens";

  // 实例属性
  name: string;
  private age: number; // 私有属性
  protected id: number; // 受保护属性（子类可访问）

  // 构造方法
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
    this.id = 123;
  }

  // 实例方法
  introduce(): string {
    return `I'm ${this.name}, ${this.age} years old`;
  }

  // 类方法
  static createAnonymous(): Person {
    return new Person("Anonymous", 0);
  }

  // 静态方法
  static isAdult(age: number): boolean {
    return age >= 18;
  }

  // Getter
  get age(): number {
    return this.age;
  }

  // Setter
  set age(value: number) {
    if (value < 0) {
      throw new Error("Age cannot be negative");
    }
    this.age = value;
  }
}

// 继承
class Student extends Person {
  grade: string;

  constructor(name: string, age: number, grade: string) {
    super(name, age);
    this.grade = grade;
  }

  introduce(): string {
    const base = super.introduce();
    return `${base}, Grade: ${this.grade}`;
  }
}

// 简写语法（推荐）
class PersonShort {
  static species = "Homo sapiens";

  constructor(
    public name: string,
    private age: number,
    protected id: number = 123
  ) {}

  introduce(): string {
    return `I'm ${this.name}, ${this.age} years old`;
  }
}
```

## ⚠️ 差异与常见陷阱

### 1. 访问修饰符

```typescript
class Example {
  publicProp: string; // 公开（默认）
  private privateProp: string; // 私有（类内部访问）
  protected protectedProp: string; // 受保护（类及子类访问）
  readonly readonlyProp: string; // 只读（初始化后不可修改）

  constructor() {
    this.publicProp = "public";
    this.privateProp = "private";
    this.protectedProp = "protected";
    this.readonlyProp = "readonly";
  }
}
```

| 修饰符 | Python                | TypeScript  |
| ------ | --------------------- | ----------- |
| 公开   | 默认                  | `public`    |
| 私有   | `_prefix` 或 `__name` | `private`   |
| 受保护 | 无原生                | `protected` |
| 只读   | `@property` 无 setter | `readonly`  |

> **记忆口诀**：`private` 是私家花园（只有自己能进），`protected` 是家族产业（子孙也能进），`public` 是公共广场（谁都能进）。

### 2. 抽象类与接口

```typescript
// 抽象类
abstract class Animal {
  abstract makeSound(): void; // 抽象方法

  move(): void {
    console.log("Moving...");
  }
}

class Dog extends Animal {
  makeSound(): void {
    console.log("Woof!");
  }
}

// 接口（TypeScript 特有）
interface Drawable {
  draw(): void;
  color?: string; // 可选属性
}

interface Measurable {
  getArea(): number;
}

// 实现多个接口
class Circle implements Drawable, Measurable {
  color: string = "red";

  constructor(private radius: number) {}

  draw(): void {
    console.log(`Drawing circle with radius ${this.radius}`);
  }

  getArea(): number {
    return Math.PI * this.radius ** 2;
  }
}
```

> **什么时候用抽象类 vs 接口？**
>
> - 抽象类：多个类需要共享实现代码（比如 Animal 都会 move）
> - 接口：只需要定义契约，不关心实现（比如 Drawable 可以是任何能画的东西）

### 3. 构造函数的简写

```typescript
// 传统写法
class Point {
  x: number;
  y: number;

  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }
}

// TypeScript 简写（自动创建属性）
class PointShort {
  constructor(
    public x: number,
    public y: number
  ) {}
}

// 使用
const p = new PointShort(3, 4);
console.log(p.x); // 3
```

### 4. this 绑定陷阱

这是 Python 开发者最容易踩的坑：

```typescript
// ❌ 错误：this 丢失
class Counter {
  count = 0;

  increment() {
    console.log(this.count); // 正常
  }

  delayedIncrement() {
    setTimeout(function () {
      console.log(this.count); // undefined！
    }, 100);
  }
}

// ✅ 正确：箭头函数保持 this
class CounterFixed {
  count = 0;

  delayedIncrement() {
    setTimeout(() => {
      console.log(this.count); // 0，正确！
    }, 100);
  }
}

// ✅ 另一种写法：类属性箭头函数
class CounterArrow {
  count = 0;

  // 箭头函数作为类属性，自动绑定 this
  increment = () => {
    console.log(this.count);
  };
}
```

> **关键区别**：
>
> - Python 的 `self` 是显式传参，总是指向实例
> - JavaScript/TypeScript 的 `this` 是隐式的，取决于调用方式
> - 箭头函数没有自己的 `this`，继承外部上下文

## 💬 开发者对话

🐍 Python 开发者："TypeScript 的 this 为什么这么诡异？在 Python 里 self 永远指向实例，但这里有时候是 undefined。"

🦀 TS 开发者："这是 JavaScript 的历史遗留问题。this 的值取决于函数怎么被调用，而不是定义在哪里。"

🐍 Python 开发者："那对象方法里用箭头函数可以解决吗？"

🦀 TS 开发者："可以，但要小心。对象字面量的方法用箭头函数，this 可能还是不对。最安全的做法是在类里用类属性箭头函数，或者用 `.bind(this)`。"

---
🐍 Python 开发者："TypeScript 的接口和抽象类有什么区别？Python 只有抽象类。"

🦀 TS 开发者："接口定义契约，抽象类可以包含实现。一个类只能继承一个抽象类，但可以实现多个接口。"

🐍 Python 开发者："就像 Python 的 Protocol 和 ABC？"

🦀 TS 开发者："差不多！接口更像 Protocol，抽象类更像 ABC。接口的好处是可以在运行时检查 `instanceof`，而且可以实现多个。"

---
🐍 Python 开发者："构造函数简写太方便了，但 public/private/protected 放在参数里会不会很难读？"

🦀 TS 开发者："习惯了就好。这叫参数属性，相当于声明属性 + 构造函数赋值一步完成。对于简单的类很清爽，复杂逻辑还是分开写更清晰。"

## 🤔 练习

### 练习 1：预测输出

看看这段代码，先预测输出结果，再展开答案：

```typescript
class User {
  name: string;
  private score = 0;

  constructor(name: string) {
    this.name = name;
  }

  addScore(points: number) {
    this.score += points;
  }

  getScore() {
    return this.score;
  }

  delayedAdd(points: number) {
    setTimeout(function () {
      this.addScore(points);
    }, 0);
  }

  delayedAddFixed(points: number) {
    setTimeout(() => {
      this.addScore(points);
    }, 0);
  }
}

const user = new User("Alice");
user.delayedAdd(10);
user.delayedAddFixed(20);
console.log(user.getScore());
```

<details>
<summary>答案</summary>

输出：`20`

解释：

1. `delayedAdd(10)` 里的普通函数中，`this` 不是 User 实例，所以 `this.addScore` 会报错（或 `this` 是 undefined）
2. `delayedAddFixed(20)` 使用箭头函数，`this` 正确绑定到 User 实例，所以 score 变成 20

> **记住**：箭头函数锁定 `this`，普通函数的 `this` 动态变化。

</details>

### 练习 2：找错误

这段代码有一个错误，找出并修复：

```typescript
class BankAccount {
  private balance: number;

  constructor(initialBalance: number) {
    // 这里忘记初始化了
  }

  deposit(amount: number) {
    this.balance += amount;
  }

  getBalance() {
    return this.balance;
  }
}

const account = new BankAccount(100);
console.log(account.getBalance());
```

<details>
<summary>答案</summary>

错误：构造函数没有给 `balance` 赋值。

修复方式 1（传统写法）：

```typescript
constructor(initialBalance: number) {
  this.balance = initialBalance;
}
```

修复方式 2（参数属性简写）：

```typescript
class BankAccount {
  constructor(private balance: number) {}
  // ...
}
```

TypeScript 会检测到 `balance` 声明了但没有赋值，编译时报错。

</details>

### 练习 3：代码翻译

将以下 Python 代码转换为 TypeScript：

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    def __init__(self, color: str):
        self.color = color

    @abstractmethod
    def area(self) -> float:
        pass

    def describe(self) -> str:
        return f"A {self.color} shape"

class Rectangle(Shape):
    def __init__(self, color: str, width: float, height: float):
        super().__init__(color)
        self.width = width
        self.height = height

    def area(self) -> float:
        return self.width * self.height
```

<details>
<summary>答案</summary>

```typescript
abstract class Shape {
  constructor(protected color: string) {}

  abstract area(): number;

  describe(): string {
    return `A ${this.color} shape`;
  }
}

class Rectangle extends Shape {
  constructor(
    color: string,
    private width: number,
    private height: number
  ) {
    super(color);
  }

  area(): number {
    return this.width * this.height;
  }
}
```

</details>

## 🎓 面试追问

1. **TypeScript 的 `private` 和 `protected` 有什么区别？**
   - 答：`private` 只能在类内部访问，`protected` 可以在类和子类中访问。

2. **什么时候用抽象类，什么时候用接口？**
   - 答：抽象类用于共享实现，接口用于定义契约。一个类只能继承一个抽象类，但可以实现多个接口。

3. **如何解决 this 绑定丢失的问题？**
   - 答：三种方式：
     - 使用箭头函数（推荐，最简洁）
     - 在构造函数里 `.bind(this)`
     - 使用类属性箭头函数 `method = () => {}`
