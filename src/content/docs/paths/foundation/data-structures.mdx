## 🎯 为什么这个很重要

你在 Python 里用 `dict` 用得好好的，转到 TypeScript 后：

```typescript
const user = { name: "Alice" };
user["age"] = 25; // ❌ 类型检查报错！
```

或者更崩溃的：

```typescript
const scores = {};
scores.math = 90; // ❌ 还是不行！
scores["english"] = 85; // ❌ 也不行！
```

为什么？TypeScript 说：`{}` 类型上不存在属性 `math`。

**真实痛点**：

1. Python 的 `dict` 想加什么键就加什么，TypeScript 的 Object 却要提前定义好结构
2. Python 的 `tuple` 只是"不可变列表"，TypeScript 的 tuple 却会检查每个位置的类型
3. Python 一个 `dict` 搞定所有键值对场景，TypeScript 却要分 Object 和 Map

数据结构是代码的基石，理解差异能让你少踩很多坑。

---
## 💡 用类比理解数据结构

把数据结构想象成不同的容器：

| 数据结构   | 类比       | 特点                                 |
| ---------- | ---------- | ------------------------------------ |
| **Array**  | 书架       | 可以放书、取书、加书，灵活可变       |
| **Object** | 档案柜     | 每个抽屉有固定标签，不能随便加新抽屉 |
| **Map**    | 万能储物柜 | 什么都能存，钥匙可以是任何类型       |
| **Set**    | 集邮册     | 每张邮票只放一次，重复的不要         |
| **Tuple**  | 定制盒子   | 固定格子，每个格子放什么早就设计好了 |

> **关键差异**：Python 的 `dict` 像万能储物柜（Map），什么键都能加。TypeScript 的 Object 像档案柜，结构要先设计好。

## 📚 Python 回顾

```python
# List - 有序可变序列（书架）
numbers = [1, 2, 3, 4, 5]
numbers.append(6)
numbers[0] = 10

# Dictionary - 键值对（万能储物柜）
user = {
    "name": "Alice",
    "age": 25,
    "email": "alice@example.com"
}
print(user["name"])

# 随时加新键？没问题！
user["phone"] = "123-456-7890"  # ✅ Python 不在乎

# Tuple - 有序不可变序列（定制盒子）
point = (3, 4)
# point[0] = 5  # 错误！不能修改

# Set - 无序不重复集合（集邮册）
unique_numbers = {1, 2, 3, 3, 3}  # {1, 2, 3}
```

---
## 🔧 TypeScript 等价写法

```typescript
// Array - 有序可变序列（书架）
const numbers: number[] = [1, 2, 3, 4, 5];
numbers.push(6);
numbers[0] = 10;

// Object - 键值对（档案柜，结构固定）
const user: { name: string; age: number; email: string } = {
  name: "Alice",
  age: 25,
  email: "alice@example.com",
};
console.log(user.name); // 注意：使用点号访问，不是方括号

// 想加新字段？需要先在类型里定义
interface UserWithPhone {
  name: string;
  age: number;
  email: string;
  phone?: string; // 可选属性
}
const userWithPhone: UserWithPhone = { name: "Alice", age: 25, email: "alice@example.com" };
userWithPhone.phone = "123-456-7890"; // ✅ 现在可以了

// Map - 真正的万能储物柜，键可以是任意类型
const userMap = new Map<string, any>();
userMap.set("name", "Alice");
userMap.set("age", 25);
userMap.set(123, "numeric key"); // 数字也能当键
console.log(userMap.get("name"));

// Tuple - 固定长度、固定类型的数组（定制盒子）
const point: [number, number] = [3, 4];
// point[0] = "hello";  // ❌ 错误！类型不匹配
// point.push(5);       // 不推荐，虽然可以运行但破坏 tuple 语义

// Set - 无序不重复集合（集邮册）
const uniqueNumbers = new Set([1, 2, 3, 3, 3]);
console.log(uniqueNumbers); // Set {1, 2, 3}
```

## ⚠️ 差异与常见陷阱

### 1. 字典 vs Object vs Map：怎么选？

| 场景           | Python | TypeScript | 原因                   |
| -------------- | ------ | ---------- | ---------------------- |
| 已知结构的数据 | `dict` | `Object`   | 类型安全，IDE 提示友好 |
| 动态键值对     | `dict` | `Map`      | 键可以是任意类型       |
| 键是数字/对象  | `dict` | `Map`      | Object 键只能是 string |

```python
# Python：一个 dict 搞定一切
person = {"name": "Bob"}
person["age"] = 30              # ✅ 随时加
person["hobbies"] = ["reading"] # ✅ 值类型随意
```

```typescript
// TypeScript：需要区分场景

// ❌ 错误示范：用 Object 存动态键
const person: { name: string } = { name: "Bob" };
person.age = 30; // 编译报错！

// ✅ 正确方式 1：提前定义接口
interface Person {
  name: string;
  age?: number;
  hobbies?: string[];
}
const person1: Person = { name: "Bob" };
person1.age = 30; // ✅ 可以

// ✅ 正确方式 2：用 Map 存动态数据
const person2 = new Map<string, any>();
person2.set("name", "Bob");
person2.set("age", 30);
person2.set("hobbies", ["reading"]);
```

### 2. 数组/列表操作速查表

| 操作         | Python          | TypeScript          | 备注            |
| ------------ | --------------- | ------------------- | --------------- |
| 添加元素     | `append()`      | `push()`            | 名字不同        |
| 移除最后一个 | `pop()`         | `pop()`             | 一样            |
| 获取长度     | `len(list)`     | `array.length`      | 函数 vs 属性    |
| 切片         | `list[1:4]`     | `array.slice(1, 4)` | Python 用语法糖 |
| 包含检查     | `x in list`     | `array.includes(x)` | 方法不同        |
| 查找索引     | `list.index(x)` | `array.indexOf(x)`  | 名字类似        |

```typescript
const arr = [1, 2, 3, 4, 5];

// 切片
const slice1 = arr.slice(1, 4); // [2, 3, 4]（索引 1 到 3）
const slice2 = arr.slice(2); // [3, 4, 5]（索引 2 到末尾）
const slice3 = arr.slice(-2); // [4, 5]（最后两个）

// 包含检查
const hasThree = arr.includes(3); // true

// 查找索引
const index = arr.indexOf(3); // 2
```

### 3. 解构赋值

```python
# Python 解构（unpacking）
point = (3, 4)
x, y = point

first, *rest = [1, 2, 3, 4, 5]  # first=1, rest=[2,3,4,5]
```

```typescript
// TypeScript 解构
const point: [number, number] = [3, 4];
const [x, y] = point;

const [first, ...rest] = [1, 2, 3, 4, 5]; // first=1, rest=[2,3,4,5]

// 对象解构（Python 没有直接对应）
const user = { name: "Alice", age: 25, city: "NYC" };
const { name, age } = user; // 提取 name 和 age
const { name: userName } = user; // 重命名
```

### 4. Tuple 的严格性

Python 的 tuple 只是"不可变列表"，TypeScript 的 tuple 却会检查每个位置的类型：

```python
# Python：tuple 只是不可变，不检查位置类型
point = (3, 4)
name_age = ("Alice", 25)
# 这两个 tuple 用法一样，只是内容不同
```

```typescript
// TypeScript：tuple 每个位置都有固定类型
const point: [number, number] = [3, 4];
const nameAge: [string, number] = ["Alice", 25];

// 混用会报错
const wrong: [number, number] = [3, "four"]; // ❌ 类型错误！
```

## 💬 开发者对话

🐍 **Python 开发者**："TypeScript 为什么分 Object 和 Map？Python 一个 `dict` 搞定！"

🦀 **TS 开发者**："Object 适合结构已知的数据，Map 适合动态键值对。比如用户信息用 Object，缓存系统用 Map。"

🐍 **Python 开发者**："那我 Python 里的 `user["new_field"] = value` 怎么转？"

🦀 **TS 开发者**："两个选择：要么提前在接口里定义 `new_field?: type`，要么用 `Map<string, any>`。Object 要先设计好结构，不能随便加字段。"

🐍 **Python 开发者**："tuple 也有区别？不都是不可变吗？"

🦀 **TS 开发者**："Python 的 tuple 是'不可变列表'，TypeScript 的 tuple 是'固定长度、每位置固定类型'。TS 会检查 `const point: [number, number]` 的第一个位置必须是数字。"

🐍 **Python 开发者**："那 Set 呢？看着差不多。"

🦀 **TS 开发者**："Set 的用法差不多，都是去重。只是 Python 用 `{1, 2, 3}` 语法，TypeScript 用 `new Set([1, 2, 3])`。"

## 🤔 试试看

### 预测输出

下面代码会输出什么？先想想，再看答案。

```typescript
const obj: { name: string } = { name: "Alice" };
const map = new Map<string, string>();
map.set("name", "Alice");

console.log(Object.keys(obj).length); // ?
console.log(map.size); // ?

obj["age"] = "25"; // 会报错吗？
map.set("age", "25"); // 会报错吗？
```

<details>
<summary>答案</summary>

```typescript
console.log(Object.keys(obj).length); // 1
console.log(map.size); // 1

obj["age"] = "25"; // ❌ 编译错误！类型 "{ name: string }" 上不存在属性 "age"
map.set("age", "25"); // ✅ 正常运行
```

**原因**：Object 的类型在定义时就固定了，不能动态添加属性。Map 是专门为动态键值对设计的。

</details>

### 找错误

下面代码有问题，你能找到吗？

```typescript
// 统计单词出现次数
function countWords(words: string[]): { [key: string]: number } {
  const counts = {};
  for (const word of words) {
    counts[word] = (counts[word] || 0) + 1;
  }
  return counts;
}
```

<details>
<summary>提示</summary>

看 `counts` 的初始化和类型定义。

</details>

<details>
<summary>答案</summary>

```typescript
// ❌ 错误：counts 类型是 {}，没有索引签名
const counts = {}; // 类型推断为 {}，不能有属性

// ✅ 修复方式 1：添加类型注解
const counts: { [key: string]: number } = {};

// ✅ 修复方式 2：使用 Record 类型
const counts: Record<string, number> = {};

// ✅ 修复方式 3：使用 Map
const counts = new Map<string, number>();
for (const word of words) {
  counts.set(word, (counts.get(word) || 0) + 1);
}
```

**关键点**：`const counts = {}` 的类型是 `{}`（空对象），不是"可以有任何属性的对象"。要支持动态键，需要显式声明索引签名。

</details>

## 📝 练习

### 代码翻译

将以下 Python 代码转换为 TypeScript：

```python
# 创建一个用户列表
users = [
    {"name": "Alice", "age": 25},
    {"name": "Bob", "age": 30}
]

# 找到年龄大于 26 的用户
older_users = [u for u in users if u["age"] > 26]

# 创建一个不重复的邮箱集合
emails = {"alice@example.com", "bob@example.com", "alice@example.com"}
```

<details>
<summary>答案</summary>

```typescript
// 定义接口
interface User {
  name: string;
  age: number;
}

// 创建一个用户列表
const users: User[] = [
  { name: "Alice", age: 25 },
  { name: "Bob", age: 30 },
];

// 找到年龄大于 26 的用户
const olderUsers: User[] = users.filter((u) => u.age > 26);

// 创建一个不重复的邮箱集合
const emails = new Set<string>(["alice@example.com", "bob@example.com", "alice@example.com"]);
console.log(emails); // Set {2} {"alice@example.com", "bob@example.com"}
```

</details>

| -------- | -------------- | ------------------- |

## 🎓 面试追问

1. **什么时候用 Object，什么时候用 Map？**
   - 答：Object 用于结构已知的数据（如用户信息、配置对象），Map 用于动态键值对或键不是 string 的情况（如缓存、事件订阅）。

2. **TypeScript 的 `readonly` 数组是什么？**
   - 答：`readonly number[]` 表示数组内容不可修改，类似 Python tuple 的不可变性。

3. **如何让 Object 支持动态键？**
   - 答：使用索引签名 `{ [key: string]: any }` 或 `Record<string, any>` 类型。
