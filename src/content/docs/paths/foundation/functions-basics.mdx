## 🎯 为什么这个很重要

你写了一个处理订单的函数，Python 版本运行得好好的。移植到 TypeScript 后，编译器突然报错：

```
Parameter 'discount' implicitly has an 'any' type.
Argument of type 'string | undefined' is not assignable to parameter of type 'number'.
```

你盯着屏幕发愣："明明 Python 里一行代码就能搞定，为什么 TypeScript 这么啰嗦？"

问题出在这里：Python 的函数像一台"自由咖啡机"——什么都往里扔，运行时才告诉你"这不能放茶叶"。TypeScript 的函数则是一台"带说明书的咖啡机"——机器上贴着标签，只接受咖啡豆，投茶叶会被拒绝。

这意味着什么？**你在写代码时就能发现问题，而不是等到用户下单时才崩溃。**

## 💡 核心概念

### 函数 = 一台机器

把函数想象成一台机器：

| 部分                 | Python             | TypeScript             |
| -------------------- | ------------------ | ---------------------- |
| **输入口（参数）**   | 什么都行，不贴标签 | 贴标签：`name: string` |
| **输出口（返回值）** | 不知道出什么       | 标明：`: string`       |
| **质检员**           | 没有，运行时看运气 | 编译器，写代码就检查   |

```typescript
// TypeScript 的函数"说明书"
function greet(name: string): string {
  //        ↑ 输入类型      ↑ 输出类型
  return `Hello, ${name}!`;
}
```

### 箭头函数 = 速溶咖啡

普通函数像手冲咖啡——步骤完整，适合复杂场景。箭头函数像速溶咖啡——快速便捷，适合简单操作。

```typescript
// 手冲咖啡：完整写法
function double(n: number): number {
  return n * 2;
}

// 速溶咖啡：箭头函数，一行搞定
const double = (n: number): number => n * 2;
```

> **什么时候用箭头函数？**
>
> - 回调函数：`numbers.map(n => n * 2)`
> - 简单计算：`(a, b) => a + b`
> - 需要保持外部 `this` 时（后面会讲）

## 📚 Python 回顾

```python
# 基本函数定义
def greet(name):
    return f"Hello, {name}!"

# 带类型提示的函数（Python 3.5+）
def add(a: int, b: int) -> int:
    return a + b

# 默认参数
def power(base: int, exponent: int = 2) -> int:
    return base ** exponent

# 可变参数
def sum_all(*numbers: int) -> int:
    return sum(numbers)

# 关键字参数
def create_user(name: str, **kwargs) -> dict:
    user = {"name": name}
    user.update(kwargs)
    return user

# Lambda 表达式
square = lambda x: x ** 2
numbers = [1, 2, 3, 4]
squared = list(map(square, numbers))
```

## 🦀 TypeScript 等价写法

```typescript
// 基本函数定义
function greet(name: string): string {
  return `Hello, ${name}!`;
}

// 带类型注解的函数
function add(a: number, b: number): number {
  return a + b;
}

// 默认参数
function power(base: number, exponent: number = 2): number {
  return Math.pow(base, exponent);
}

// 剩余参数（类似 Python *args）
function sumAll(...numbers: number[]): number {
  return numbers.reduce((sum, n) => sum + n, 0);
}

// 对象参数（替代 Python **kwargs）
interface UserOptions {
  age?: number;
  email?: string;
}

function createUser(name: string, options?: UserOptions): { name: string } & UserOptions {
  return { name, ...options };
}

// 箭头函数（类似 Lambda）
const square = (x: number): number => x * x;
const numbers: number[] = [1, 2, 3, 4];
const squared = numbers.map(square);

// 箭头函数的简写
const doubled = numbers.map((n) => n * 2);

// 无返回值函数
function logMessage(message: string): void {
  console.log(message);
}
```

## ⚠️ 差异与常见陷阱

### 1. 函数声明方式

```typescript
// ✅ 函数声明（有提升，可以在定义前调用）
add(1, 2); // 正常工作

function add(a: number, b: number): number {
  return a + b;
}

// ❌ 函数表达式（无提升，必须先定义后使用）
subtract(1, 2); // 报错！

const subtract = function (a: number, b: number): number {
  return a - b;
};

// ✅ 箭头函数（推荐用于回调）
const multiply = (a: number, b: number): number => a * b;
```

### 2. this 绑定（重点！）

这是 Python 开发者最容易踩的坑。

```typescript
const person = {
  name: "Alice",
  age: 25,

  // ✅ 普通函数：this 指向调用者（person）
  greetRegular: function () {
    console.log(`Hello, I'm ${this.name}`);
  },

  // ❌ 箭头函数作为对象方法：this 继承外部上下文
  greetArrow: () => {
    console.log(`Hello, I'm ${this.name}`); // this.name 是 undefined！
  },

  // ✅ 简写方法（推荐）
  greet() {
    console.log(`Hello, I'm ${this.name}`);
  },
};
```

**记忆口诀**：

- 箭头函数 = 锁定 `this`，继承定义时的上下文
- 普通函数 = 动态 `this`，指向调用者

### 3. 可选参数与默认值

```typescript
// 可选参数（必须放在最后）
function greet(name: string, greeting?: string): string {
  if (greeting) {
    return `${greeting}, ${name}!`;
  }
  return `Hello, ${name}!`;
}

// 带默认值（更推荐）
function greet(name: string, greeting: string = "Hello"): string {
  return `${greeting}, ${name}!`;
}

// 混合使用：可选参数 + 默认值
function createUser(
  name: string,
  age?: number,
  isActive: boolean = true
): { name: string; age?: number; isActive: boolean } {
  return { name, age, isActive };
}
```

### 4. 函数重载 = 多功能咖啡机

同一台机器，选择不同模式，输出不同产品。TypeScript 支持函数重载，Python 不支持。

```typescript
// 重载签名：定义多种输入输出组合
function process(value: string): string;
function process(value: number): number;

// 实现签名：处理所有情况
function process(value: string | number): string | number {
  if (typeof value === "string") {
    return value.toUpperCase();
  }
  return value * 2;
}

process("hello"); // 返回 "HELLO"（string 类型）
process(42); // 返回 84（number 类型）
```

## 📊 优势对比

| 特性       | Python     | TypeScript |
| ---------- | ---------- | ---------- |
| 类型检查   | 运行时     | 编译时     |
| 默认参数   | 支持       | 支持       |
| 关键字参数 | `**kwargs` | 对象解构   |
| 函数重载   | 不支持     | 支持       |
| 箭头函数   | Lambda     | 原生支持   |
| 返回值推断 | 无         | 有         |

## 💬 开发者对话

> 🐍 **Python 开发者**：为什么 TypeScript 要写这么多类型？Python 一行 `def add(a, b): return a + b` 就完了。
>
> 🦀 **TypeScript 开发者**：等你传了字符串进去，运行时报错就知道了。TypeScript 在写代码时就告诉你"参数类型不对"。
>
> 🐍 **Python 开发者**：好吧...但这个 `this` 绑定太诡异了！为什么箭头函数里的 `this` 是 `undefined`？
>
> 🦀 **TypeScript 开发者**：箭头函数没有自己的 `this`，它会"继承"外部的 `this`。对象方法里用箭头函数，`this` 指向的是定义时的上下文，不是对象本身。
>
> 🐍 **Python 开发者**：所以...对象方法不要用箭头函数？
>
> 🦀 **TypeScript 开发者**：没错！对象方法用普通函数或简写方法。箭头函数适合回调，比如 `array.map(x => x * 2)`。
>
> 🐍 **Python 开发者**：那 `**kwargs` 呢？TypeScript 怎么接收任意关键字参数？
>
> 🦀 **TypeScript 开发者**：TypeScript 用**接口**定义参数结构。虽然不能像 Python 那样"随意"，但好处是编辑器会提示你有哪些字段可以用。

## 🤔 试试看

### 预测输出

先别运行代码，猜猜输出是什么？

```typescript
const counter = {
  count: 0,

  increment: () => {
    this.count++;
    console.log(this.count);
  },

  reset() {
    this.count = 0;
    console.log(this.count);
  },
};

counter.increment();
counter.reset();
```

<details>
<summary>点击查看答案</summary>

输出：

```
NaN
0
```

**解析**：

- `increment` 是箭头函数，`this` 不指向 `counter` 对象，而是外部上下文。`this.count` 是 `undefined`，`undefined++` 是 `NaN`。
- `reset` 是普通函数简写，`this` 正确指向 `counter`，所以能访问 `this.count`。

**修正方法**：把 `increment` 改成普通函数：

```typescript
increment() {
  this.count++;
  console.log(this.count);
}
```

</details>

### 找错误

这段代码有什么问题？

```typescript
class Button {
  label: string;

  constructor(label: string) {
    this.label = label;
  }

  // 点击回调
  handleClick = () => {
    console.log(`Clicked: ${this.label}`);
  };
}

const btn = new Button("Submit");
const handler = btn.handleClick;
handler(); // 输出什么？
```

<details>
<summary>点击查看答案</summary>

其实这段代码是**正确的**！输出 `Clicked: Submit`。

为什么？这里用的是**类属性箭头函数**（`handleClick = () => {}`），不是方法。箭头函数在构造时就绑定了 `this`，所以即使 `handler` 被单独调用，`this` 仍然指向 `Button` 实例。

这正是在类中处理事件回调的推荐方式！

</details>

## ✍️ 练习

将以下 Python 代码转换为 TypeScript：

```python
def calculate_area(shape, **params):
    if shape == "circle":
        radius = params.get("radius", 0)
        return 3.14159 * radius ** 2
    elif shape == "rectangle":
        width = params.get("width", 0)
        height = params.get("height", 0)
        return width * height
    return 0

# 使用
area1 = calculate_area("circle", radius=5)
area2 = calculate_area("rectangle", width=4, height=6)
```

<details>
<summary>答案</summary>

```typescript
interface CircleParams {
  radius: number;
}

interface RectangleParams {
  width: number;
  height: number;
}

type ShapeParams = CircleParams | RectangleParams;

function calculateArea(shape: "circle" | "rectangle", params: ShapeParams): number {
  if (shape === "circle" && "radius" in params) {
    return Math.PI * params.radius ** 2;
  } else if (shape === "rectangle" && "width" in params && "height" in params) {
    return params.width * params.height;
  }
  return 0;
}

// 使用
const area1 = calculateArea("circle", { radius: 5 });
const area2 = calculateArea("rectangle", { width: 4, height: 6 });
```

</details>

## 🎓 面试追问

1. **箭头函数和普通函数的区别？**
   - 箭头函数没有自己的 `this`，继承外部上下文
   - 箭头函数不能作为构造函数（不能 `new`）
   - 箭头函数没有 `arguments` 对象

2. **什么时候用箭头函数，什么时候用普通函数？**
   - 回调函数：箭头函数
   - 对象方法：普通函数
   - 需要动态 `this`：普通函数
   - 类事件处理：类属性箭头函数（绑定 `this`）

3. **函数重载在 TypeScript 中如何实现？有什么限制？**
   - 先写多个重载签名（只声明类型）
   - 最后写实现签名（处理所有情况）
   - 限制：重载签名必须与实现签名兼容
