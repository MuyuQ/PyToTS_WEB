## 场景与问题

Python 习惯动态类型，变量随手定义、类型随意改。TypeScript 要求编码时明确类型，刚开始可能觉得麻烦，但能省去很多调试时间。

## Python 回顾

Python 中变量类型是动态的：

```python
name = "Alice"           # str
age = 25                 # int
height = 1.75            # float
is_student = True        # bool

# 可以改变类型（虽然不建议）
age = "25 years old"     # 合法
```

## TypeScript 等价写法

TypeScript 有两种写法：显式声明或类型推断。

```typescript
// 显式声明
let name: string = "Alice";
let age: number = 25;
let height: number = 1.75;
let isStudent: boolean = true;

// 类型推断（更常用）
let name2 = "Alice"; // TypeScript 推断为 string
let age2 = 25; // 推断为 number

// const 用于常量
const PI = 3.14159;
const GREETING = "Hello";
```

默认使用 `const`，确定需要重新赋值时才用 `let`。

## 差异与常见陷阱

### 1. 变量声明关键字

| Python   | TypeScript | 说明         |
| -------- | ---------- | ------------ |
| 直接赋值 | `let`      | 可变变量     |
| 无对应   | `const`    | 不可变的常量 |

```typescript
let count = 0;
count = 1; // ✓ 合法

const MAX_SIZE = 100;
MAX_SIZE = 200; // ✗ 报错：const 不能重新赋值
```

### 2. 数字类型统一

Python 区分 `int` 和 `float`，TypeScript 统一为 `number`：

```typescript
let x: number = 5;
let y: number = 5.0; // 都是 number
```

### 3. 类型不能随意改变

```typescript
let value = "hello";
value = 123; // ✗ 报错

// 需要多种类型时使用联合类型
let flexible: string | number = "hello";
flexible = 123; // ✓ 合法
```

## 优势对比

| 特性     | Python             | TypeScript |
| -------- | ------------------ | ---------- |
| 类型声明 | 可选（Type Hints） | 可选但推荐 |
| 类型检查 | 运行时             | 编译时     |
| 自动补全 | 有限               | 强大       |
| 重构安全 | 较低               | 高         |

Python 类型检查在运行时，TypeScript 在编译时检查，提前发现错误。

## 练习

将以下 Python 代码转换为 TypeScript：

```python
username = "developer"
score = 95.5
is_passed = True
attempts = 3
```

<details>
<summary>答案</summary>

```typescript
const username: string = "developer";
let score: number = 95.5;
const isPassed: boolean = true; // Python 的 True 变成小写 true
let attempts: number = 3;
```

</details>

## 面试追问

**什么时候用 `let`，什么时候用 `const`？**

默认使用 `const`，只有确定需要重新赋值时才用 `let`。

**TypeScript 的 `any` 类型是什么？应该避免吗？**

`any` 表示任意类型，会绕过类型检查。能不用就不用，如果真不知道类型，用 `unknown` 更安全。
