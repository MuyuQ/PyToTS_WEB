# 变量与数据类型

## 场景与问题

作为 Python 开发者，你已经熟悉动态类型的便利。但 TypeScript 是静态类型的，这意味着你需要在编码时就考虑类型。本节将帮助你建立类型思维的基础。

## Python 回顾

在 Python 中，变量类型是动态的：

```python
# Python 变量不需要声明类型
name = "Alice"           # str
age = 25                 # int
height = 1.75            # float
is_student = True        # bool

# 可以随时改变类型
age = "25 years old"     # 合法，但不推荐
```

## TypeScript 等价写法

在 TypeScript 中，你需要显式声明或使用类型推断：

```typescript
// 显式类型声明
let name: string = "Alice";
let age: number = 25;
let height: number = 1.75;
let isStudent: boolean = true;

// 类型推断（推荐）
let name2 = "Alice";      // TypeScript 推断为 string
let age2 = 25;            // TypeScript 推断为 number

// const 用于不可重新赋值的变量
const PI = 3.14159;
const GREETING = "Hello";
```

## 差异与常见陷阱

### 1. 变量声明关键字

| Python | TypeScript | 说明 |
|--------|-----------|------|
| 直接赋值 | `let` | 可重新赋值的变量 |
| 无对应 | `const` | 不可重新赋值的常量 |

```typescript
// TypeScript
let count = 0;
count = 1;           // ✓ 合法

const MAX_SIZE = 100;
MAX_SIZE = 200;      // ✗ 错误！不能重新赋值 const
```

### 2. 数字类型统一

```python
# Python 区分 int 和 float
x: int = 5
y: float = 5.0
```

```typescript
// TypeScript 统一为 number
let x: number = 5;
let y: number = 5.0;  // 都是 number 类型
```

### 3. 类型一旦确定不能改变

```typescript
let value = "hello";
value = 123;         // ✗ 错误！不能将 number 赋给 string

// 解决方案：使用联合类型
let flexible: string | number = "hello";
flexible = 123;      // ✓ 合法
```

## 优势对比

| 特性 | Python | TypeScript |
|------|--------|-----------|
| 类型声明 | 可选（Type Hints） | 可选但推荐 |
| 类型检查 | 运行时 | 编译时 |
| 自动补全 | 有限 | 强大 |
| 重构安全 | 较低 | 高 |

## 练习

将以下 Python 代码转换为 TypeScript：

```python
username = "developer"
score = 95.5
is_passed = True
attempts = 3
```

<details>
<summary>答案</summary>

```typescript
const username: string = "developer";
let score: number = 95.5;
const isPassed: boolean = True;  // 注意：Python 的 True 变成 TypeScript 的 true
let attempts: number = 3;
```
</details>

## 面试追问

1. **什么时候用 `let`，什么时候用 `const`？**
   - 答：默认使用 `const`，只有当变量需要重新赋值时才用 `let`。

2. **TypeScript 的 `any` 类型是什么？应该避免吗？**
   - 答：`any` 表示任意类型，会绕过类型检查。应尽量避免，使用 `unknown` 替代。
