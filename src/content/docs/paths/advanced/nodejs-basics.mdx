## 场景与问题

Python 开发者习惯使用 `open()`、`os.path` 和 `pathlib` 处理文件系统操作。迁移到 Node.js 后，需要理解回调风格、Promise 化 API、Buffer 和 Stream 等概念。这些是构建命令行工具、后端服务和处理大文件的基础。

## 核心概念

Node.js 是 JavaScript 运行时，提供了丰富的内置模块用于系统编程。与 Python 的标准库相比，Node.js 的 API 设计更强调异步非阻塞。

| 功能       | Python               | Node.js          |
| ---------- | -------------------- | ---------------- |
| 文件读写   | `open()`, `io`       | `fs` 模块        |
| 路径处理   | `os.path`, `pathlib` | `path` 模块      |
| 环境变量   | `os.environ`         | `process.env`    |
| 命令行参数 | `sys.argv`           | `process.argv`   |
| 进程退出   | `sys.exit()`         | `process.exit()` |
| 哈希计算   | `hashlib`            | `crypto` 模块    |

## 文件操作

### 基础读写

Python 使用 `open()` 函数进行文件操作，Node.js 提供 `fs` 模块。

#### Python 实现

```python
# 读取文件
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()

# 写入文件
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("Hello, World!")

# 追加写入
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("New log entry\n")

# 逐行读取
with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())
```

#### Node.js 实现（回调风格）

```typescript
import fs from "fs";

// 读取文件（异步回调）
fs.readFile("data.txt", "utf-8", (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});

// 写入文件（异步回调）
fs.writeFile("output.txt", "Hello, World!", "utf-8", (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log("File written");
});

// 追加写入
fs.appendFile("log.txt", "New log entry\n", (err) => {
  if (err) console.error(err);
});
```

#### Node.js 实现（Promise 风格）

```typescript
import fs from "fs/promises";

// 读取文件
async function readFileExample(): Promise<void> {
  try {
    const content = await fs.readFile("data.txt", "utf-8");
    console.log(content);
  } catch (err) {
    console.error(err);
  }
}

// 写入文件
async function writeFileExample(): Promise<void> {
  try {
    await fs.writeFile("output.txt", "Hello, World!", "utf-8");
    console.log("File written");
  } catch (err) {
    console.error(err);
  }
}

// 检查文件是否存在
async function checkFile(path: string): Promise<boolean> {
  try {
    await fs.access(path, fs.constants.R_OK);
    return true;
  } catch {
    return false;
  }
}

// 获取文件信息
async function getFileInfo(path: string): Promise<void> {
  const stats = await fs.stat(path);
  console.log(`Size: ${stats.size}`);
  console.log(`Is directory: ${stats.isDirectory()}`);
  console.log(`Modified: ${stats.mtime}`);
}
```

### 同步 vs 异步

Node.js 提供三种风格的文件操作 API。

```typescript
import fs from "fs";
import fsPromises from "fs/promises";

// 1. 同步版本（阻塞事件循环）
try {
  const data = fs.readFileSync("data.txt", "utf-8");
  console.log(data);
} catch (err) {
  console.error(err);
}

// 2. 回调版本（非阻塞，旧风格）
fs.readFile("data.txt", "utf-8", (err, data) => {
  if (err) throw err;
  console.log(data);
});

// 3. Promise 版本（推荐，非阻塞，现代风格）
async function readWithPromise(): Promise<void> {
  const data = await fsPromises.readFile("data.txt", "utf-8");
  console.log(data);
}
```

**何时使用同步 API：**

- 应用启动时加载配置文件
- 命令行工具脚本
- 性能要求不高的场景

**何时使用异步 API：**

- Web 服务器处理请求
- 高并发场景
- 处理大量文件 I/O

## Buffer 和 Stream

### Buffer 类型

Buffer 是 Node.js 处理二进制数据的核心类型，类似 Python 的 `bytes`。

#### Python 实现

```python
# 创建 bytes
data = b"Hello"
data = bytes([72, 101, 108, 108, 111])

# 字符串与 bytes 转换
text = "你好"
encoded = text.encode("utf-8")  # b'\xe4\xbd\xa0\xe5\xa5\xbd'
decoded = encoded.decode("utf-8")  # "你好"

# 十六进制转换
hex_str = encoded.hex()  # "e4bda0e5a5bd"
from_hex = bytes.fromhex("e4bda0e5a5bd")

# 字节操作
data = bytearray(b"Hello")
data[0] = 74  # 修改字节
print(bytes(data))  # b"Jello"

# 读取二进制文件
with open("image.png", "rb") as f:
    binary_data = f.read()
```

#### Node.js 实现

```typescript
// 创建 Buffer
const buf1 = Buffer.from("Hello", "utf-8");
const buf2 = Buffer.from([72, 101, 108, 108, 111]);
const buf3 = Buffer.alloc(10); // 分配 10 字节，填充 0
const buf4 = Buffer.allocUnsafe(10); // 分配 10 字节，不初始化（更快但有旧数据）

// 字符串与 Buffer 转换
const text = "你好";
const encoded = Buffer.from(text, "utf-8"); // <Buffer e4 bd a0 e5 a5 bd>
const decoded = encoded.toString("utf-8"); // "你好"

// 十六进制转换
const hexStr = encoded.toString("hex"); // "e4bda0e5a5bd"
const fromHex = Buffer.from("e4bda0e5a5bd", "hex");

// Buffer 操作
const buf = Buffer.from("Hello");
console.log(buf[0]); // 72 (H 的 ASCII 码)
console.log(buf.length); // 5
console.log(buf.toString("base64")); // "SGVsbG8="

// Buffer 拼接
const combined = Buffer.concat([Buffer.from("Hello"), Buffer.from(" World")]);
console.log(combined.toString()); // "Hello World"

// Buffer 切片
const slice = buf.subarray(0, 3);
console.log(slice.toString()); // "Hel"

// Buffer 比较
const a = Buffer.from("abc");
const b = Buffer.from("abc");
console.log(a.equals(b)); // true
console.log(a.compare(Buffer.from("abd"))); // -1 (a < abd)
```

### Stream 基础

Stream 是 Node.js 处理流式数据的抽象，适合处理大文件或实时数据。

#### 四种 Stream 类型

| 类型      | 说明                     | Python 对比   |
| --------- | ------------------------ | ------------- |
| Readable  | 可读流                   | `io.StringIO` |
| Writable  | 可写流                   | 文件写入模式  |
| Duplex    | 双工流（可读可写）       | Socket 连接   |
| Transform | 转换流（读写时转换数据） | 管道处理      |

#### Python 实现（文件流）

```python
# 分块读取大文件
def process_large_file(filepath, chunk_size=8192):
    with open(filepath, "rb") as f:
        while chunk := f.read(chunk_size):
            yield chunk  # 使用生成器模拟流

# 流式处理（使用管道）
import sys
for line in sys.stdin:
    processed = line.upper()
    sys.stdout.write(processed)

# StringIO 内存流
from io import StringIO
buffer = StringIO()
buffer.write("Hello")
buffer.write(" World")
print(buffer.getvalue())  # "Hello World"
```

#### Node.js 实现

```typescript
import fs from "fs";
import { Transform, pipeline } from "stream";

// 1. Readable Stream - 读取大文件
const readStream = fs.createReadStream("large-file.txt", {
  encoding: "utf-8",
  highWaterMark: 64 * 1024, // 64KB 缓冲区
});

readStream.on("data", (chunk: string | Buffer) => {
  console.log(`Received ${chunk.length} bytes`);
});

readStream.on("end", () => {
  console.log("File reading completed");
});

readStream.on("error", (err: Error) => {
  console.error("Error:", err);
});

// 2. Writable Stream - 写入文件
const writeStream = fs.createWriteStream("output.txt");

writeStream.write("Hello\n");
writeStream.write("World\n");
writeStream.end(); // 必须调用 end() 来关闭流

writeStream.on("finish", () => {
  console.log("Write completed");
});

// 3. Transform Stream - 数据转换
const upperCaseTransform = new Transform({
  transform(
    chunk: Buffer | string,
    encoding: BufferEncoding,
    callback: (error?: Error | null, data?: Buffer | string) => void
  ): void {
    const text = chunk.toString().toUpperCase();
    callback(null, text);
  },
});

// 4. 使用 pipe 连接流
fs.createReadStream("input.txt").pipe(upperCaseTransform).pipe(fs.createWriteStream("output.txt"));

// 5. 使用 pipeline（更好的错误处理）
pipeline(
  fs.createReadStream("input.txt"),
  upperCaseTransform,
  fs.createWriteStream("output.txt"),
  (err) => {
    if (err) {
      console.error("Pipeline failed:", err);
    } else {
      console.log("Pipeline succeeded");
    }
  }
);

// 6. 异步迭代器（Node.js 10+）
async function processStream(): Promise<void> {
  const stream = fs.createReadStream("large-file.txt", "utf-8");
  for await (const chunk of stream) {
    console.log(`Processing chunk of ${chunk.length} chars`);
  }
}
```

### Stream 实际应用：复制大文件

#### Python 实现

```python
import shutil

# 简单复制
shutil.copy2("source.mp4", "dest.mp4")

# 流式复制（适合超大文件）
def copy_large_file(src, dst, chunk_size=1024 * 1024):
    with open(src, "rb") as f_src, open(dst, "wb") as f_dst:
        while chunk := f_src.read(chunk_size):
            f_dst.write(chunk)

# 使用管道模式
def transform_copy(src, dst, transform_func):
    with open(src, "rb") as f_src, open(dst, "wb") as f_dst:
        while chunk := f_src.read(8192):
            transformed = transform_func(chunk)
            f_dst.write(transformed)
```

#### Node.js 实现

```typescript
import fs from "fs";
import { pipeline } from "stream";
import { createGzip, createGunzip } from "zlib";
import { promisify } from "util";

const pipelineAsync = promisify(pipeline);

// 简单复制
async function copyFile(src: string, dst: string): Promise<void> {
  await pipelineAsync(fs.createReadStream(src), fs.createWriteStream(dst));
  console.log("Copy completed");
}

// 复制并压缩
async function compressFile(src: string, dst: string): Promise<void> {
  await pipelineAsync(fs.createReadStream(src), createGzip(), fs.createWriteStream(dst));
  console.log("Compress completed");
}

// 解压文件
async function decompressFile(src: string, dst: string): Promise<void> {
  await pipelineAsync(fs.createReadStream(src), createGunzip(), fs.createWriteStream(dst));
  console.log("Decompress completed");
}

// 使用示例
compressFile("data.txt", "data.txt.gz").catch(console.error);
```

### 进程控制

#### Python 实现

```python
import sys
import signal
import time

# 退出进程
sys.exit(0)  # 正常退出
sys.exit(1)  # 错误退出

# 退出码
print(f"Exit code: {sys.exit()}")  # 这行不会执行

# 信号处理
def handle_signal(signum, frame):
    print(f"Received signal {signum}")
    sys.exit(0)

signal.signal(signal.SIGINT, handle_signal)
signal.signal(signal.SIGTERM, handle_signal)

# 进程信息
print(f"PID: {os.getpid()}")
print(f"PPID: {os.getppid()}")  # 父进程 ID
```

#### Node.js 实现

```typescript
// 退出进程
process.exit(0); // 正常退出
process.exit(1); // 错误退出

// 退出码
console.log(`Exit code: ${process.exitCode}`); // 获取/设置退出码
process.exitCode = 1; // 设置退出码（不立即退出）

// 进程信息
console.log(`PID: ${process.pid}`);
console.log(`Platform: ${process.platform}`);
console.log(`Node version: ${process.version}`);
console.log(`Architecture: ${process.arch}`);

// 内存使用
const memory = process.memoryUsage();
console.log(`RSS: ${memory.rss / 1024 / 1024} MB`); // 常驻内存
console.log(`Heap: ${memory.heapUsed / 1024 / 1024} MB`);

// CPU 使用
const cpuUsage = process.cpuUsage();
console.log(`User: ${cpuUsage.user} μs`);
console.log(`System: ${cpuUsage.system} μs`);

// 运行时间
console.log(`Uptime: ${process.uptime()} seconds`);

// 信号处理
process.on("SIGINT", () => {
  console.log("Received SIGINT (Ctrl+C)");
  process.exit(0);
});

process.on("SIGTERM", () => {
  console.log("Received SIGTERM");
  // 清理资源
  process.exit(0);
});

// 未捕获异常处理
process.on("uncaughtException", (error: Error) => {
  console.error("Uncaught Exception:", error);
  process.exit(1);
});

// 未处理的 Promise 拒绝
process.on("unhandledRejection", (reason: unknown, promise: Promise<unknown>) => {
  console.error("Unhandled Rejection at:", promise, "reason:", reason);
});
```

## 常用 Node.js API

### crypto 模块（哈希与加密）

#### Python 实现

```python
import hashlib
import hmac
import secrets
import base64

# 计算哈希
data = "Hello, World!"
hash_md5 = hashlib.md5(data.encode()).hexdigest()
hash_sha256 = hashlib.sha256(data.encode()).hexdigest()
hash_sha512 = hashlib.sha512(data.encode()).hexdigest()

print(f"MD5: {hash_md5}")
print(f"SHA256: {hash_sha256}")
print(f"SHA512: {hash_sha512}")
```

```typescript
async function hashPassword(password: string): Promise<string> {
  const salt = crypto.randomBytes(16).toString("hex");
  return new Promise((resolve, reject) => {
    crypto.pbkdf2(password, salt, 100000, 64, "sha512", (err, derivedKey) => {
      if (err) reject(err);
      resolve(`${salt}:${derivedKey.toString("hex")}`);
    });
  });
}

async function verifyPassword(password: string, stored: string): Promise<boolean> {
  const [salt, hash] = stored.split(":");
  return new Promise((resolve, reject) => {
    crypto.pbkdf2(password, salt, 100000, 64, "sha512", (err, derivedKey) => {
      if (err) reject(err);
      resolve(hash === derivedKey.toString("hex"));
    });
  });
}
```

## 差异与常见陷阱

### 文件路径问题

```typescript
// ✗ 错误：硬编码路径分隔符
const wrongPath = "data" + "\\" + "file.txt"; // Windows only
const alsoWrong = "data" + "/" + "file.txt"; // Unix only

// ✓ 正确：使用 path.join
import path from "path";
const correctPath = path.join("data", "file.txt"); // 自动处理分隔符

// ✗ 错误：使用相对路径访问脚本目录外的文件
const configPath = "../config.json"; // 相对于当前工作目录

// ✓ 正确：使用 __dirname 构建绝对路径
const configPath = path.join(__dirname, "..", "config.json");
```

### 编码问题

```typescript
// ✗ 错误：不指定编码，得到 Buffer
const data = fs.readFileSync("text.txt"); // 返回 Buffer

// ✓ 正确：指定编码得到字符串
const data = fs.readFileSync("text.txt", "utf-8");

// ✗ 错误：处理二进制文件时用 utf-8
const image = fs.readFileSync("image.png", "utf-8"); // 会损坏数据

// ✓ 正确：二进制文件使用 Buffer
const image = fs.readFileSync("image.png"); // 返回 Buffer
```

### 异步陷阱

```typescript
// ✗ 错误：混用回调和 async/await
fs.readFile("file.txt", (err, data) => {
  if (err) throw err;
  return data; // 无法正确返回
});

// ✓ 正确：统一使用 Promise
const data = await fs.promises.readFile("file.txt", "utf-8");

// ✗ 错误：忘记 await
async function bad() {
  const promise = fs.promises.readFile("file.txt", "utf-8");
  console.log(promise); // Promise { <pending> }
}

// ✓ 正确：使用 await
async function good() {
  const data = await fs.promises.readFile("file.txt", "utf-8");
  console.log(data); // 实际内容
}

// ✗ 错误：循环中的异步操作
for (const file of files) {
  fs.promises.readFile(file); // 并发执行，不等待
}

// ✓ 正确：顺序执行
for (const file of files) {
  const data = await fs.promises.readFile(file);
  console.log(data);
}

// ✓ 正确：并发执行并等待所有完成
const results = await Promise.all(files.map((file) => fs.promises.readFile(file, "utf-8")));
```

### Buffer 拼接陷阱

```typescript
// ✗ 错误：字符串拼接 Buffer
const chunks: Buffer[] = [];
stream.on("data", (chunk: Buffer) => {
  chunks.push(chunk);
});
stream.on("end", () => {
  const result = chunks.join(""); // 错误！会调用 toString()
});

// ✓ 正确：使用 Buffer.concat

  const result = Buffer.concat(chunks).toString("utf-8");

```

### process.exit() 陷阱

```typescript
// ✗ 错误：process.exit() 立即终止，不等待异步操作

process.on("SIGINT", () => gracefulShutdown("SIGINT"));
```

## 练习

实现一个简单的文件监控工具，监控指定文件的变更并记录日志。

要求：

1. 监控文件内容变化
2. 记录变更时间戳
3. 支持异步处理
4. 使用 Stream 处理大文件

```typescript
// 期望用法
const watcher = new FileWatcher("data.txt", "changes.log");
watcher.start();
// 当 data.txt 变更时，在 changes.log 中记录时间戳
```

<details>
<summary>答案</summary>

```typescript
import fs from "fs/promises";
import fsSync from "fs";
import path from "path";

class FileWatcher {
  private targetFile: string;
  private logFile: string;
  private watcher: fsSync.FSWatcher | null = null;

  constructor(targetFile: string, logFile: string) {
    this.targetFile = path.resolve(targetFile);
    this.logFile = path.resolve(logFile);
  }

  async start(): Promise<void> {
    // 初始化日志文件
    await fs.appendFile(this.logFile, `=== Watching ${this.targetFile} ===\n`);

    // 使用 fs.watch 监控文件
    this.watcher = fsSync.watch(
      this.targetFile,
      async (eventType: string, filename: string | null) => {
        if (eventType === "change") {
          await this.logChange();
        }
      }
    );

    this.watcher.on("error", (err: Error) => {
      console.error(`Watcher error: ${err.message}`);
    });

    console.log(`Watching ${this.targetFile}...`);
  }

  private async logChange(): Promise<void> {
    const timestamp = new Date().toISOString();
    const logEntry = `[${timestamp}] File changed\n`;

    try {
      await fs.appendFile(this.logFile, logEntry);
      console.log(`Change logged at ${timestamp}`);
    } catch (err) {
      console.error(`Failed to log change: ${err}`);
    }
  }

  stop(): void {
    if (this.watcher) {
      this.watcher.close();
      this.watcher = null;
      console.log("Watcher stopped");
    }
  }
}

// 使用示例
const watcher = new FileWatcher("data.txt", "changes.log");
watcher.start();

// 优雅退出
process.on("SIGINT", () => {
  watcher.stop();
  process.exit(0);
});
```

## 面试追问

1. **fs.readFile 和 fs.createReadStream 有什么区别？什么时候用哪个？**
   - 答：`readFile` 一次性读取整个文件到内存，适合小文件。`createReadStream` 创建可读流，分块读取，适合大文件。当文件超过可用内存或处理流式数据时，必须用 Stream。

2. **Buffer.alloc 和 Buffer.allocUnsafe 有什么区别？**
   - 答：`alloc` 初始化内存为 0，更安全但稍慢。`allocUnsafe` 不初始化，可能包含旧数据，更快但有安全风险。生产环境推荐 `alloc`，性能敏感场景确认数据会被覆盖时可用 `allocUnsafe`。

3. **process.nextTick 和 setImmediate 有什么区别？**
   - 答：`process.nextTick` 在当前操作完成后、事件循环继续前执行，优先级更高。`setImmediate` 在下一个事件循环迭代中执行。`nextTick` 可能导致 I/O 饥饿，大量使用时要注意。

4. **如何在 Node.js 中实现大文件复制而不占用过多内存？**
   - 答：使用 Stream 和 pipe 或 pipeline：

   ```typescript
   import { pipeline } from "stream";
   import fs from "fs";

   await pipeline(fs.createReadStream("large.mp4"), fs.createWriteStream("copy.mp4"));
   ```

   这样数据分块流动，内存中只保留当前处理的块。

5. **Node.js 的 event loop 和 Python 的 asyncio 有什么区别？**
   - 答：Node.js 事件循环是运行时内置的，所有 I/O 操作默认非阻塞。Python 的 asyncio 需要显式使用 async/await 和事件循环，同步 I/O 会阻塞整个循环。Node.js 的非阻塞 I/O 是系统级的，Python 需要 asyncio 兼容的库。
