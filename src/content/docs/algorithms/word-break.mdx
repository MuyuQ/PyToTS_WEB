## 问题描述

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。请你判断是否可以利用字典中出现的单词拼接出 `s`。

**注意：**不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

## 思路分析

动态规划：

- `dp[i]`：前 i 个字符能否被拆分
- 状态转移：`dp[i] = dp[j] and s[j:i] in wordDict` 对于所有 `0 <= j < i`
- 初始化：`dp[0] = True`

优化：可以用 Trie 或哈希集合加速子串查询

## 复杂度分析

- **时间复杂度**：O(n²)，最坏情况需要检查所有子串
- **空间复杂度**：O(n)，dp数组 + 哈希集合

    ```python
    # 单词拆分 - 动态规划 + 哈希集合
    # 算法: Word Break
    # 时间复杂度: O(n²), 最坏情况需要检查所有子串
    # 空间复杂度: O(n), dp数组 + 哈希集合

    def word_break(s: str, wordDict: list[str]) -> bool:
        # 将字典列表转换为哈希集合，O(1) 时间复杂度查询单词是否存在
        # 比列表的 O(n) 查询效率高很多
        word_set = set(wordDict)

        # 获取目标字符串的长度
        n = len(s)

        # dp[i] 表示前 i 个字符（s[0..i-1]）能否被拆分成字典中的单词
        # 初始化为 False，表示初始时所有位置都不可达
        dp = [False] * (n + 1)  # 索引 0~n

        # 边界条件：空字符串可以被拆分（不需要任何单词）
        dp[0] = True

        # 外层循环：遍历每个位置 i（1到n）
        # 计算 dp[i]：前 i 个字符能否被拆分
        for i in range(1, n + 1):
            # 内层循环：遍历所有可能的分割点 j（0到i-1）
            # 尝试将 s[0..i-1] 分割成 s[0..j-1] 和 s[j..i-1]
            for j in range(i):
                # 状态转移条件：
                # 1. dp[j] 为 True：前 j 个字符可以被拆分
                # 2. s[j:i] 在字典中：剩余子串是一个有效单词
                # 两个条件都满足时，前 i 个字符可以被拆分
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True  # 标记为可拆分
                    break  # 提前退出：找到一个有效分割即可，不需要继续检查

        # 返回 dp[n]：整个字符串能否被拆分
        return dp[n]

    # 测试用例
    print(word_break("leetcode", ["leet", "code"])) # True ("leet"+"code")
    print(word_break("applepenapple", ["apple", "pen"])) # True ("apple"+"pen"+"apple")
    print(word_break("catsandog", ["cats", "dog", "sand", "and", "cat"])) # False
    print(word_break("aaaaaaa", ["aaaa", "aaa"])) # True ("aaaa"+"aaa" 或 "aaa"+"aaaa")
    ```

## 面试变体

1. **所有拆分方式**：返回所有可能的拆分方式
2. **最少单词数**：使用最少单词的拆分
3. **最长单词**：使用最长单词优先的策略

## 常见错误

1. 忘记初始化 `dp[0] = True`
2. 子串范围写错（应该是 s[j:i] 不是 s[j:i-1]）
3. 内层循环可以提前退出但没有退出
