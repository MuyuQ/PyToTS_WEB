## 问题描述

有 `n` 个网络节点，标记为 `1` 到 `n`。

给定列表 `times`，其中 `times[i] = (u, v, w)` 表示从节点 `u` 到节点 `v` 的信号传输时间为 `w`。

给定起始节点 `k`，求从节点 `k` 发出信号后，所有节点都收到信号所需的最短时间。如果无法让所有节点都收到信号，返回 `-1`。

## 思路分析

Dijkstra 最短路径算法：

1. **构建邻接表**：将边列表转换为图结构
2. **初始化距离数组**：源节点距离为 0，其他为无穷大
3. **优先队列**：使用最小堆，每次取出距离最小的节点
4. **松弛操作**：如果通过当前节点到达邻居的距离更短，更新距离
5. **结果判断**：所有节点的最大距离即为答案

核心思想：

- 贪心策略：每次选择当前距离最小的未访问节点
- 优先队列保证高效获取最小值
- 不能处理负权边（需要 Bellman-Ford）

## 复杂度分析

- **时间复杂度**：O(E log V)，每条边最多入堆一次，堆操作 O(log V)
- **空间复杂度**：O(V + E)，邻接表和距离数组

## 面试变体

1. **返回最短路径**：记录前驱节点，重构从源到任意节点的路径
2. **多个源节点**：多源最短路径，使用虚拟源节点或 Floyd-Warshall
3. **负权边**：使用 Bellman-Ford 或 SPFA 算法
4. **K 短路径**：A\* 算法或 Yen's algorithm
5. **动态边权**：根据时间变化的边权，需要修改松弛条件

## 常见错误

1. **节点编号问题**：题目节点从 1 开始，数组索引从 0 开始，需要调整
2. **堆中重复节点**：同一节点可能多次入堆，需要 visited 集合
3. **返回值类型**：Python 用 `float('inf')`，TypeScript 用 `Infinity`
4. **优先队列方向**：Dijkstra 需要最小堆，Prim 最小生成树也是最小堆
5. **忘记检查不可达**：最后要判断是否有节点距离仍为无穷大

## 扩展：为什么 Dijkstra 不能处理负权边？

Dijkstra 基于贪心策略，一旦确定某节点的最短距离就不会再更新。但负权边可能导致后续找到更短的路径。

```
    A --(2)--> B
    |         |
   (1)      (-3)
    |         |
    v         v
    C --(1)--> D

从 A 出发：
- Dijkstra 先处理 A->B(2)，标记 B 的距离为 2
- 然后处理 A->C(1)，C 的距离为 1
- 但实际上 A->C->D->B 的距离是 1+1+(-3)=-1，比 2 更短！
```

负权边场景应使用 Bellman-Ford 算法。
