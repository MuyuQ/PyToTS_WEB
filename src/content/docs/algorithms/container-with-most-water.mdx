## 问题描述

给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

## 思路分析

双指针法：

- 初始时左右指针分别指向数组两端
- 计算当前面积：`(right - left) * min(height[left], height[right])`
- 移动高度较小的那个指针（因为移动较高的指针不会增加面积）
- 更新最大面积

## 复杂度分析

- **时间复杂度**：O(n)，两个指针向中间移动
- **空间复杂度**：O(1)，只使用常数额外空间

    # 盛最多水的容器 - 双指针贪心算法
    # 时间复杂度: O(n), 空间复杂度: O(1)
    # n 为高度数组的长度

    def max_area(height: list[int]) -> int: # left: 左指针, 初始指向数组最左端 # right: 右指针, 初始指向数组最右端
    left, right = 0, len(height) - 1 # max_water: 记录最大盛水量, 初始为 0
    max_water = 0

        # 当左右指针未相遇时继续循环
        while left < right:
            # width: 容器宽度, 等于两指针之间的距离
            width = right - left
            # current_height: 容器高度, 由较短的那条线决定 (木桶效应)
            current_height = min(height[left], height[right])
            # current_area: 当前容器的盛水量 = 宽度 × 高度
            current_area = width * current_height

            # 更新最大盛水量
            max_water = max(max_water, current_area)

            # 贪心策略: 移动高度较小的指针
            # 原因: 容器高度由较短的线决定, 移动较高的线只会让宽度减小
            # 而高度不会增加 (仍然受限于较短的线), 面积必然减小
            # 只有移动较短的线, 才有可能找到更高的线来增加面积
            if height[left] < height[right]:
                left += 1   # 左侧线较短, 左指针右移
            else:
                right -= 1  # 右侧线较短 (或相等), 右指针左移

        # 返回找到的最大盛水量
        return max_water

    # 测试用例 1: [1,8,6,2,5,4,8,3,7] -> 最大面积为 49 (索引 1 和 8 的线)

    print(max_area([1, 8, 6, 2, 5, 4, 8, 3, 7])) # 输出: 49

    # 测试用例 2: [1,1] -> 只有一条宽为 1 高为 1 的容器

    print(max_area([1, 1])) # 输出: 1

    # 测试用例 3: [4,3,2,1,4] -> 最大面积为 16 (两端索引 0 和 4 的线)

    print(max_area([4, 3, 2, 1, 4])) # 输出: 16

    // n 为高度数组的长度

    function maxArea(height: number[]): number {
    // left: 左指针, 初始指向数组最左端
    let left = 0;
    // right: 右指针, 初始指向数组最右端
    let right = height.length - 1;
    // maxWater: 记录最大盛水量, 初始为 0
    let maxWater = 0;

        // 当左右指针未相遇时继续循环
        while (left < right) {
            // width: 容器宽度, 等于两指针之间的距离
            const width = right - left;
            // currentHeight: 容器高度, 由较短的那条线决定 (木桶效应)
            const currentHeight = Math.min(height[left], height[right]);
            // currentArea: 当前容器的盛水量 = 宽度 × 高度

            maxWater = Math.max(maxWater, currentArea);

            // 贪心策略: 移动高度较小的指针
            // 原因: 容器高度由较短的线决定, 移动较高的线只会让宽度减小
            // 而高度不会增加 (仍然受限于较短的线), 面积必然减小
            // 只有移动较短的线, 才有可能找到更高的线来增加面积
            if (height[left] < height[right]) {
                left++;   // 左侧线较短, 左指针右移
            } else {
                right--;  // 右侧线较短 (或相等), 右指针左移
            }
        }

        // 返回找到的最大盛水量
        return maxWater;

    }

    console.log(maxArea([1, 1])); // 输出: 1

## 面试变体

1. **三指针问题**：使用三条线构成的容器
2. **最多k个容器**：找出 k 个容器的最大总容量
3. **最小容量**：找出容量至少为某个值的容器对

## 常见错误

1. 使用 `max` 而不是 `min` 计算高度
2. 移动指针时逻辑错误，应该移动高度较小的
3. 使用双重循环导致 O(n²) 复杂度
