## 问题描述

给定两个字符串 `s` 和 `t` 。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果不存在符合条件的子串，则返回空字符串 `""`。

## 思路分析

滑动窗口 + 哈希表：

1. 用哈希表记录 `t` 中每个字符需要的次数
2. 右指针扩展窗口，包含字符
3. 当窗口包含所有需要的字符时，尝试收缩左指针
4. 更新最小窗口

## 复杂度分析

- **时间复杂度**：O(m + n)，m 和 n 分别是两个字符串的长度
- **空间复杂度**：O(m)，哈希表存储字符频率

    # right: 滑动窗口右边界, 遍历字符串 s 的每个字符

        # 如果该字符是 t 中需要的 (need[char] > 0)

// 时间复杂度: O(m + n), 空间复杂度: O(m)
// m 为字符串 t 的长度, n 为字符串 s 的长度

function minWindow(s: string, t: string): string {
// need: Map 记录 t 中每个字符需要的次数 (正数表示还缺, 负数表示多余)
const need = new Map<string, number>();
// 初始化 need: 遍历 t, 统计每个字符的出现次数
for (const char of t) {
need.set(char, (need.get(char) || 0) + 1);
}

    // missing: 还需要匹配的字符总数, 初始为 t 的长度
    let missing = t.length;
    // left: 滑动窗口左边界, 初始为 0
    let left = 0;
    // minLen: 记录最小窗口长度, 初始为正无穷
    let minLen = Infinity;
    // minStart: 记录最小窗口的起始位置
    let minStart = 0;

    // right: 滑动窗口右边界, 遍历字符串 s 的每个字符
    for (let right = 0; right < s.length; right++) {
        // 获取当前遍历到的字符
        const char = s[right];
        // 获取该字符在 need 中的计数值
        const count = need.get(char);

        // 如果该字符是 t 中需要的 (count > 0)
        // 说明这个字符对匹配有贡献, missing 减 1
        if (count !== undefined && count > 0) {
            missing--;
        }
        // 无论是否需要, 都将该字符的计数减 1
        // 正数表示还需要, 0 表示刚好, 负数表示窗口中该字符有多余
        need.set(char, (count || 0) - 1);

        // 当 missing == 0 时, 说明窗口已包含 t 的所有字符
        // 此时尝试收缩左边界以找到更小的窗口
        while (missing === 0) {
            // 如果当前窗口长度更小, 更新最小窗口记录
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;  // 更新最小长度
                minStart = left;             // 更新最小窗口起始位置
            }

            // 准备移动左边界, 获取左边界字符
            const leftChar = s[left];
            // 获取左边界字符的计数值
            const leftCount = need.get(leftChar)!;
            // 将左边界字符的计数加 1
            need.set(leftChar, leftCount + 1);
            // 如果加 1 后计数变为正数, 说明该字符不再满足需求
            // missing 加 1, 窗口不再包含所有 t 的字符
            if (leftCount + 1 > 0) {
                missing++;
            }
            // 左边界右移一位
            left++;
        }
    }

    // 如果 minLen 仍为无穷大, 说明没找到符合条件的子串, 返回空字符串
    // 否则返回 s 中从 minStart 开始、长度为 minLen 的子串
    return minLen === Infinity ? "" : s.substring(minStart, minStart + minLen);

}

## 面试变体

1. **包含排列**：找到包含 `t` 的某个排列的最小子串
2. **最多k个不同字符**：包含所有 `t` 中字符，且窗口内最多 k 个不同字符
3. **优化**：使用数组代替哈希表优化常数

## 常见错误

1. 混淆 `need` 数组的更新逻辑
2. 忘记处理不存在的情况
3. 边界条件错误，子串长度计算错误
