## 问题描述

一条包含字母 `A-Z` 的消息通过以下映射进行了**编码**：

```
'A' -> "1"
'B' -> "2"
...
'Z' -> "26"
```

要**解码**一条已编码的消息，所有数字必须分组，然后按上面的映射方法反向映射回字母（可能存在多种方式）。

例如，`"11106"` 可以映射为：

- `"AAJF"` ，将消息分组为 `(1 1 10 6)`
- `"KJF"` ，将消息分组为 `(11 10 6)`

注意，消息不能分组为 `(1 11 06)` ，因为 `"06"` 不能映射为 `"F"` ，这是由于 `"6"` 和 `"06"` 在映射中并不等价。

给你一个只含数字的**非空**字符串 `s`，请计算并返回**解码**方法的**总数**。

## 思路分析

动态规划：

- `dp[i]`：解码前 i 个字符的方法数
- 状态转移：
  - 单独解码 `s[i-1]`（如果是 '1'-'9'）：`dp[i] += dp[i-1]`
  - 和前一个字符组合解码（如果 `10 <= s[i-2:i] <= 26`）：`dp[i] += dp[i-2]`

## 复杂度分析

- **时间复杂度**：O(n)，遍历字符串
- **空间复杂度**：O(1)，只使用两个变量

## Python 实现

```python
# 解码方法 - 动态规划 (空间优化)
# 算法: Decode Ways
# 时间复杂度: O(n), 遍历字符串一次
# 空间复杂度: O(1), 只使用两个变量存储状态

def num_decodings(s: str) -> int: # 边界条件1：空字符串，无法解码 # 边界条件2：字符串以 '0' 开头，无法解码（0没有对应字母）
if not s or s[0] == '0':
return 0

    # 获取字符串长度
    n = len(s)

    # 空间优化：只保留前两个状态
    # prev2 对应 dp[i-2]，解码前 i-2 个字符的方法数
    # 初始值 1 表示空字符串有1种解码方式（什么都不做）
    prev2 = 1  # dp[0] = 1 (空字符串)

    # prev1 对应 dp[i-1]，解码前 i-1 个字符的方法数
    # 初始值 1 表示第一个字符（已确认不是'0'）有1种解码方式
    prev1 = 1  # dp[1] = 1 (第一个字符)

    # 从第二个字符开始遍历（索引1）
    for i in range(1, n):
        # current 存储解码到当前位置的总方法数
        current = 0

        # 情况1：单独解码当前字符 s[i]
        # 只有当 s[i] 是 '1'-'9' 时才能单独解码
        # '0' 不能单独解码，必须和前一个字符组合
        if '1' <= s[i] <= '9':
            current += prev1  # 加上解码前 i-1 个字符的方法数

        # 情况2：和前一个字符组合解码 s[i-1:i+1]
        # 提取两位数字字符串并转换为整数
        two_digit = int(s[i-1:i+1])

        # 只有当两位数字在 10-26 范围内时才能组合解码
        # 注意：01-09 不合法（前导0），27-99 超出字母范围
        if 10 <= two_digit <= 26:
            current += prev2  # 加上解码前 i-2 个字符的方法数

        # 更新状态：向前移动一步
        prev2 = prev1  # 原来的 prev1 变成新的 prev2
        prev1 = current  # 当前计算的方法数变成新的 prev1

    # 返回解码整个字符串的方法数
    return prev1

# 测试用例

print(num_decodings("12")) # 2 ("AB"=1,2 或 "L"=12)
print(num_decodings("226")) # 3 ("BBF"=2,2,6 或 "BZ"=2,26 或 "VF"=22,6)
print(num_decodings("0")) # 0 (以0开头，无法解码)
print(num_decodings("10")) # 1 ("J"=10, 注意0不能单独解码)
print(num_decodings("27")) # 1 ("BG"=2,7, 注意27超出26不能组合)
```

## TypeScript 实现

```typescript
// 解码方法 - 动态规划 (空间优化)
// 算法: Decode Ways
// 时间复杂度: O(n), 遍历字符串一次
// 空间复杂度: O(1), 只使用两个变量存储状态

function numDecodings(s: string): number {
// 边界条件1：空字符串，无法解码
// 边界条件2：字符串以 '0' 开头，无法解码（0没有对应字母）
if (s.length === 0 || s[0] === '0') {
return 0;
}

    // 获取字符串长度
    const n = s.length;

    // 空间优化：只保留前两个状态
    // prev2 对应 dp[i-2]，解码前 i-2 个字符的方法数
    // 初始值 1 表示空字符串有1种解码方式（什么都不做）
    let prev2 = 1;  // dp[0] = 1 (空字符串)

    // prev1 对应 dp[i-1]，解码前 i-1 个字符的方法数
    // 初始值 1 表示第一个字符（已确认不是'0'）有1种解码方式
    let prev1 = 1;  // dp[1] = 1 (第一个字符)

    // 从第二个字符开始遍历（索引1）
    for (let i = 1; i < n; i++) {
        // current 存储解码到当前位置的总方法数
        let current = 0;

        // 情况1：单独解码当前字符 s[i]
        // 只有当 s[i] 是 '1'-'9' 时才能单独解码
        // '0' 不能单独解码，必须和前一个字符组合
        if (s[i] >= '1' && s[i] <= '9') {
            current += prev1;  // 加上解码前 i-1 个字符的方法数
        }

        // 情况2：和前一个字符组合解码 s[i-1:i+1]
        // 提取两位数字子串并转换为整数
        const twoDigit = parseInt(s.substring(i - 1, i + 1));

        // 只有当两位数字在 10-26 范围内时才能组合解码
        // 注意：01-09 不合法（前导0），27-99 超出字母范围
        if (twoDigit >= 10 && twoDigit <= 26) {
            current += prev2;  // 加上解码前 i-2 个字符的方法数
        }

        // 更新状态：向前移动一步
        prev2 = prev1;  // 原来的 prev1 变成新的 prev2
        prev1 = current;  // 当前计算的方法数变成新的 prev1
    }

    // 返回解码整个字符串的方法数
    return prev1;

}

// 测试用例
console.log(numDecodings("12")); // 2
console.log(numDecodings("226")); // 3
console.log(numDecodings("0")); // 0
console.log(numDecodings("10")); // 1
console.log(numDecodings("27")); // 1
```


## 面试变体

1. **带权解码**：每种解码方式有不同的权重
2. **限制解码**：某些组合不允许
3. **最长解码**：求最长解码字符串

## 常见错误

1. 忘记处理 '0' 的特殊情况
2. 两位数字范围判断错误（是 10-26 不是 1-26）
3. 忘记处理字符串以 '0' 开头的情况
