## 问题描述

给你无向**连通**图中一个节点的引用，请你返回该图的**深拷贝**（克隆）。

图中的每个节点都包含它的值 `val` 和邻居列表 `neighbors`。

测试用例格式：

- 简单起见，每个节点的值都与它的索引相同
- 例如，第一个节点值为 1，第二个节点值为 2，以此类推

## 思路分析

### 方法一：DFS

- 使用哈希表记录原节点到克隆节点的映射
- 递归克隆每个节点及其邻居

### 方法二：BFS

- 使用队列和哈希表
- 先克隆起始节点，然后逐层克隆邻居

## 复杂度分析

- **时间复杂度**：O(V + E)，V是节点数，E是边数
- **空间复杂度**：O(V)，哈希表存储克隆节点

  function cloneGraph(node: Node | null): Node | null {
  // 处理空图边界情况
  if (!node) return null;
      // Map：原节点 -> 克隆节点的映射
      // 用于避免重复克隆和检测环
      const oldToNew = new Map<Node, Node>();

      function dfs(oldNode: Node): Node {
          // 如果节点已克隆过，直接返回克隆节点
          // 这是处理环的关键：避免无限递归
          if (oldToNew.has(oldNode)) {

          oldToNew.set(oldNode, newNode);

              // 将克隆后的邻居添加到新节点的邻居列表
              newNode.neighbors.push(dfs(neighbor));
          }

          // 返回克隆完成的节点
          return newNode;
      }

      // 从起始节点开始DFS克隆
      return dfs(node);
  }

## 面试变体

1. **克隆带随机指针的链表**：类似思路，哈希表记录映射
2. **序列化和反序列化图**：将图转为字符串再还原
3. **判断图是否同构**：结构相同但值可能不同
4. **克隆有向图**：需要考虑入度和出度
5. **克隆N叉树**：简化版的图克隆

## 常见错误

1. 忘记处理环，导致无限递归
2. 浅拷贝而不是深拷贝
3. 哈希表键的类型错误（对象引用 vs 值）
4. BFS时忘记先创建节点再加入队列
